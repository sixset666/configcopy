Обработка предназначена для загрузки данных из конфигурации Бухгалтерия предприятия 1.6 в типовое отраслевое решение Раруса. Предпологается, что обмен уже ведется направления ТОР Раруса -> Бухгалтерия предприятия. При загрузке справочников проверяется соответствие в таблице соответствий. По кодам, наименованиям и другим реквизитам поиска не происходит. Поэтому необходимо позаботиться об установке соответствий. Это можно сделать выгрузив из ТОР Раруса справочники в режиме "Выгружать справочники для нахождения соответствий" и загрузить их в бухгалтерию. Также в обработке загрзки в бухгалтерию можно отредактировать визуальными средствами таблицу соответствия справочников.

Закладка "Парметры загрузки"

Параметры загрузки находятся на одноименной закладке формы обработки загрузки данных в бухгалтерию 1.6.

  • Реквизит «Контрагент организации». Предназначен для загрузки в конфигурацию «Управление рестораном» для загрузки справочника «Организации», если при загрузке используется режим обновление реквизитов существующих объектов.

      Панель "Режим загрузки"

      • Флажок «Не создавать новых контрагентов и организаций». Если при загрузке контрагента или организации не будет надйено соответствие, то выведится соответсвующее сообщение и загрузка будет остановлена.
      • Флажок «Не обновлять объекты с соответсвием ТОР Раруса -> Бухгалтерия». Если флажок включен, то при загрузке, если найдено соответствие которое было поставлено при загрузке из ТОР Раруса в Бухгалтерию предприятия, обновление происходить не будет. РЕКОМЕНДУЕТСЯ УСТАНАВЛИВАТЬ ФЛАЖОК!!!
      • Флажок «Показать загруженные (обработанные) объекты». Если флажок включен, то после загрузки объектов на экран выводится таблица с деревом загруженных объектов и их источников.
      • Флажок «Не обновлять значения реквизитов существующих объектов». Если флажок выбран, то при загрузки объекта, если с объектом уже установлено соответствие, загрузка объекта прекращается, реквизиты не изменяются.
      • Флажок «Выводить полный отчет о загрузке». Если флажок включен, то после загрузки объектов на экран выводится подробный отчет о загрузке документов.
      • Флажок «Выводить краткий отчет». Если флажок включен, то после загрузки объектов на экран выводится краткий отчет о загрузке документов – выводятся только сообщения об ошибках и о неоднозначных ситуациях.

      кладка "Таблица соответствия"

      • Параметр «Путь». Указывается файл таблицы соответствия, хранящий соответствия объектов-источников и приемников.

      При загрузке элемента справочника происходит поиск соответствия в таблице соответствий. Если соответствие не найдено, происходит поиск элемента справочника-приемника на соответствие элементу справочника-источника по наименованию или по коду. Если элемент справочника найден, то соответствие записывается в таблицу соответствий. Если флажок «Не обновлять значения реквизитов существующих объектов» установлен, то изменения реквизитов элемента справочника – приемника не происходит, в противном случае происходит перегрузка значений реквизитов из справочника – источника. Если элемент справочника не найден, создается новая группа справочника или элемент справочника, куда перегружаются значение реквизитов элемента справочника-источника.

      Закладка "Параметры по умолчанию"

      • Блок «Договора взаиморасчетов». Заполненные значения подставляются в одноименные реквизиты загружаемых договоро взаиморасчетов.
      • Блок «Контрагенты кассовых одреров». Заполненные контрагенты будут подставляться в загружаемые кассовые ордера в ТОР Раруса у документов-источников, которых в реквизитах не должен быть указан контрагент. Название полей ввода контрагентов соответствуют наименованиям видов операций кассовых ордеров - источников.
      • Блок «Контрагенты выписки». Заполненные контрагенты будут подставляться в загружаемые выписки в ТОР Раруса у документов-источников, которых в реквизитах не должен быть указан контрагент. Название полей ввода контрагентов соответствуют наименованиям видов операций платежных поручений.
      • Статья ДДС. Если при загрузки документов значение реквизита "Статья движения денежных средств" не заполнено, то он заполняется выбранным значением. Необходимо в случае, если в бухгалтерии не ведется учет по статьям ДДС.


      Правила заполнения Контрагентов и Договоров контрагентов при перегрузке Кассовых операций

      При перегрузке РКО из "Бухгалтерии предприятия" с видом операции "Взнос наличными в банк" создается два документа: РКО, отражающий операцию выдачи денежных средств из кассы организации, и Банковская выписка, отражающая зачисление денежных средств на расчетный счет организации. 

      Правила перегрузки контрагентов в зависимости от вида операции документа - источника

       - Приход денежных средств розничная выручка -> Контрагент прием розничной выручки
       - Приход денежных средств прочее, Расход денежных средств прочее - > Контрагент и Договор контрагента подставляется в соответствии с содержанием заполненных реквизитов "Субконто" в документе-источнике в Бухгалтерии. Если контрагента и договор контрагента получить из данных реквизитов не удалось, тогда выбирается Контрагент по прочему движению денежных средств из формы обработки, аналогично производится работа с договором.
       - Взнос наличными в банк, Получение наличных денежных средств в банке - > Контрагент взнос наличными в банк
       - Выплата заработной платы, Выплата заработной платы по ведомостям, Выплата депонентов - > Контрагент выплата заработной платы
       - Инкассация денежных средств - > Контрагент инкассация денежных средств
       - Выдача денежных средств подотчетнику, Возврат денежных средств подотчетником - > Контрагент выдача денежных средств подотчетнику
      При остальных операциях при загрузке документа производится попытка загрузить Контрагента из документа-источника, если загрузка не удалась, тогда подставляется Контрагент по прочему движению денежных средств. 

      Правила перегрузки договоров контрагентов 

        При операциях Приход денежных средств розничная выручка, Возврат денежных средств подотчетником, Получение наличных денежных средств в банке, Взнос наличными в банк,Выплата заработной платы, Выплата заработной платы по ведомостям, Выплата депонентов, Инкассация денежных средств, Выдача денежных средств подотчетнику, Возврат денежных средств подотчетником  -  выбирается Договор с наименованием  "Договор_<Наименование операции>" (если договор уже существует, то будет выбран существующий договор, если данного договора еще нет, то он будет создан), при остальных операциях договор перегружается из документа источника.
        

      Правила заполнения Контрагентов и Договоров контрагентов при перегрузке Платежных поручений

      Правила перегрузки контрагентов в зависимости от вида операции документа - источника

      Для операции вида Перевод средств между счетами контрагент выбирается на форме обработки, при перегрузке остальных операций Контрагент выбирается в соответствии с контрагентом документа-источника. 

      Правила перегрузки договоров контрагентов 

      Договор контрагента для операций Прочее поступление безналичных денежных средств, Прочее списание безналичных денежных средств, Перечисление заработной платы, Перевод на другой счет, Перечисление налога, Перечисление денежных средств подотчетнику - выбирается аналогично как и для кассовых операций, при остальных видах операций выбирается договор контрагента из документа-источника.
