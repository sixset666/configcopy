Обработка "АРМ администратора"

Назначение

Обработка представляет собой рабочее место администратора и предназначена для управления конфигурацией в режиме предприятия.

Относится к подсистеме: «Автоматизированные рабочие места».

Формы

  • Интерфейс администратора. Является основной формой АРМ.

    Закладки формы:

      • Пользователи. На закладке расположены два табличных поля.
        • Верхнее поле содержит список пользователей информационной базы и соответствующих им пользователей конфигурации. Данное соответствие строится на основании совпадения имени пользователя базы и кода (идентификатора)  зарегистрированного в конфигурации пользователя. (справочник «Пользователи»). Если соответствие не найдено, такой пользователь выделяется бордовым цветом. Жирным шрифтом выделяется текущий пользователь. Если у пользователя установлен реквизит "Является шаблоном", он выводится синим цветом. В колонках "Аут. ОС и "Аут. 1С" отображается, какой способ аутентификации установлен у пользователя – аутентификация Windows и 1С соответственно. Также в списке отображены доступные роли пользователя, основной интерфейс, организация и подразделение, шаблон прав, соответствующий пользователю, сотрудник  (см справочник «Сотрудники»). По всем этим параметрам можно установить отбор или сортировку списка. Пользователей можно добавлять (кнопка "Добавить" в командной панели), копировать (кнопка "Скопировать"), редактировать (кнопка "Изменить"), удалять (кнопка "Удалить"). При удалении пользователей необходимо выбрать режим удаления - пользователь базы удаляется, пользователь компании и сотрудник помечаются на удаление, у сотрудника также можно проставить флажок "уволен".
          •  Все настройки параметров пользователя производятся в форме "Форма пользователя". 
          • С помощью пункта контекстного меню "Найти в справочнике" можно найти любой элемент ссылочного типа в соответствующем ему справочнике.
          • При нажатии на кнопку «Журнал» командной панели списка пользователей происходит переключение на закладку «Журнал регистрации» с установленным отбором по выделенному пользователю в списке пользователей.
          • При нажатии на кнопку «Права» командной панели списка пользователей происходит переключение на закладку «Права и настройки», при этом тип объекта устанавливается как пользователь, а объект - выделенный пользователь в списке пользователей.
          • При нажатии на кнопку «Очистить настройки» командной панели списка пользователей происходит очистка системных настроек (таких как аутентификация ОС, пароль, сохраненные параметры, настройки списков, отчетов и обработок и т.д.) пользователя, выделенного в списке пользователей.
        • Нижнее поле содержит список активных в данный момент пользователей. Жирным шрифтом выделяется текущий пользователь. Также отображается сетевое имя компьютера пользователя, время начала сеанса, и имя приложения. Пользователей можно отключить от конфигурации (кнопка "Отключить" в командной панели) если они работают в режиме предприятия, выделив их с помощью клавиш Shift или Ctrl. При этом отключается первый найденный сеанс для каждого пользователя. Чтобы отключить всех пользователей, кроме текущего, необходимо нажать на кнопку "Отключить всех". При активизации строки списка пользователей выделяется строка сеанса этого пользователя. Чтобы найти пользователя, соответствующего сеансу, необходимо выделить его и нажать кнопку "Найти в списке пользователей" командной панели или контекстного меню.  В списке пользователей и сеансов следующие значки означают:  - у пользователя нет пароля, но установлена аутентификация Windows,  - у пользователя установлен пароль,   - у пользователя отключена аутентификация Windows  и не установлен пароль;  - отключаемый в данный момент сеанс пользователя.
      • Журнал регистрации.  Содержит список зарегистрированных событий конфигурации. Является упрощенным аналогом штатного журнала регистрации. В левой колонке отображается значок, соответствующий важности события:  - информация,  - ошибка,  - предупреждение,  -примечание. Если уставлен флаг «Автозагрузка журнала при открытии» на командной панели, то при первом открытии закладки «Журнал регистрации» происходит автоматическая загрузка журнала с установленным в прошлый раз отбором, с периодом за последние два дня. Можно установить фильтр на отображаемые события (кнопка «Отбор» в командной панели). На командную панель вынесено три параметра отбора: «Объект», «Пользователь» и «Период». После изменения параметров отбора на командной панели необходимо нажать кнопку «Обновить» для загрузки журнала с установленным отбором. При установке отбора по видам объектов или пользователю таким, по которым нет записей в журнале регистрации, загружается весь журнал за выбранный период. При двойном щелчке мышью по списку журнала в колонке «Имя пользователя» открывается форма элемента справочника «Пользователи» выделенного пользователя. При двойном щелчке мышью по списку журнала в колонке «Метаданные» открывается форма объекта, который представлен в выделенной строке.
      • Все объекты. Предоставляет быстрый доступ ко всем объектам конфигурации. В контекстном меню  можно выбрать для открытия конкретную форму объекта или вызвать справку по нему.
      • Права и настройки. Предназначена для установки прав и настроек. Права и настройки – это взаимосвязь пользователя и его возможностей переключения режимов работы Типового решения. Каждому режиму работы соответствует определенное право или настройка. В реквизите Тип объекта указывается, какую группу прав настраиваем. Объект - пользователь, организация или подразделения для которого задаются права и настройки.
        Флажки параметры отображения:
        - только по выбранному объекту - если установлен, то будут показываться только права и настройки, назначенные для группы, указанной в Типе объекта
        - отображать в дереве права - если установлен, то в дереве будут отображается права (красный кружок)
        - отображать в дереве настройки - если установлен, то в дереве будут отображается настройки (зеленый кружок)
        Кнопка Установить права по умолчанию позволяет установить значения прав и настроек по умолчанию для выбранного объекта.

        Право на выбор определенного режима работы обычному пользователю предоставляется администратором Типового решения. Обычный пользователь не может изменить ничьи права, в том числе даже свои собственные. Права могут назначаться не только для отдельных пользователей, но и для подразделений, организаций и компании в целом.

        Настройка позволяет обычному пользователю выбрать режим своей работы. Если некоторая настройка разрешена, то появляется возможность переключить соответствующий режим работы с Типовым решением.

      • Фоновые задания. Предназначенная для создания(редактирования) регламентных заданий, запуска на выполнение фоновых заданий. На странице отображен список регламентных заданий и список фоновых заданий. 

        - Список регламентных заданий. В данном списке показаны предопределенные регламентные задания, а также регламентные задания, определенные пользователем. При добавлении задания в список пользователь в отдельном окне заполняет параметры. Кнопка   позволяет создать фоновое задание для выделенного регламентного задания и запустить его на выполнение. При использовании кнопки  можно редактировать расписание регламентного задания. 

        - Список фоновых заданий. В данном списке отображается перечень фоновых заданий, созданных на основании регламентных заданий. Предоставляется возможность проследить временной интервал выполнения задания, статус, возникшие ошибки и др. Пользователю предоставлена возможность указания интервала обновления списка фоновых заданий(по умолчанию обновление не происходит), а также задать временной интервал в качестве фильтра - будут отражены только те фоновые задания, которые выполнялись в определенном временном промежутке.

  • Установка фильтра журнала регистрации. В форме настраивается фильтр на события, отображаемые в журнале регистрации. Если при закрытии формы АРМ Администратора установлен флажок «Автозагрузка журнала при открытии» на командной панели закладки "Журнал регистрации", то фильтр сохраняет свое состояние и восстанавливает его при следующем открытии.
    - Период регистрации ("Период с: по:").
    - Пользователь (поле ввода "Пользователь").
    - Компьютер (поле ввода "Компьютер").
    - Важность (список "Важность").
    - Метаданные (иерархический список "Метаданные").
     
  • Форма отбора и сортировки списка пользователей. Форма служит для отбора и сортировки пользователей списка пользователей. Вызывается при нажатии на кнопку "Установить отбор и сортировку списка" на командной панели или в контекстном меню списка пользователей. 
  • Форма удаления пользователя. Форма для настройки удаления строки пользователя.  Вызывается при нажатии на кнопку "Удалить" на командной панели или в контекстном меню списка пользователей.  При открытии формы, флажки, не имеющие смысла, отображаются недоступными. Установленные значения флажков формы сохраняются при закрытии.

Способ вызова

Открывается из обработки "Функциональная панель". Отдельно не используется.

Дополнительная информация

Форма "Интерфейс администратора" предоставляет возможности для отключения/включения сеансов пользователей.

Кнопка "Отключить" выполняет отключение выбранных сеансов пользователей (текущая строка в списке активных пользователей или выделенные строки данного списка). Отключение производится по следующей схеме:

  • задается вопрос о варианте отключении выбранных сеансов пользователей.
  • при положительном ответе ("Отключение с предупреждением пользователей") предлагается ввести текст сообщения пользователю. При отказе от ввода процесс отключения прерывается. Данное сообщение будет отображаться в отключаемом сеансе пользователя каждые 45 секунд: на 2-ий раз будет предложено завершить работу с сохранением всех редактируемых данных, на 3-ий раз сеанс будет завершен принудительно, на 5-ый раз сеанс будет завершен без сохранения редактируемых данных и не смотря на открытые модальные окна (аварийно)
  • при отрицательном ответе ("Отключение без предупреждения пользователей") будет задан вопрос с просьбой подтвердить немедленное отключение выбранных сеансов или выполнить отключение с предупреждением. При отрицательном ответе произойдет отключение выбранных сеансов с предупреждением. При положительном ответе в случае файл-серверного типа расположения ИБ отключение произойдет максимум в течение 45 секунд (то есть время проверки данных об отключениях). Отключение в данном случае применимо только для активных сеансов 1С:Предприятия. При клиент-серверном типе расположения будет выполнена попытка подключения к кластеру серверов с именем и паролем администратора кластера, центрального сервера (если они существуют) и подключение к ИБ от имени администратора с последующим немедленным (аварийным) отключением выбранных сеансов пользователей. В данном случае будут отключены все выбранные сеансы, включая Конфигуратор и внешние соединения (за исключением  текущего сеанса). Следует отметить, что для успешного выполнения немедленного отключения в клиент-серверной ИБ нужно корректно указать имена пользователей кластера, центрального сервера и ИБ в параметрах настройки на закладке "Администрирование"

    • Кнопка "Включить" разрешает работу с ИБ для  выбранных и отключенных ранее пользователей. Таким образом воспользоваться включением отдельных пользователей можно до момента отключения их сеансов. При необходимости разрешения работы для одного отключенного пользователя можно нажать кнопку удаления пользователя, снять галочку "Временно запретить работу пользователя с программой" и нажать "Выполнить".

    Кнопка "Отключить всех" выполняет отключение всех активных сеансов пользователей по той же схеме, что и кнопка "Отключить".

    Кнопка "Включить всех" разрешает работу с ИБ для всех ранее отключенных пользователей вне зависимости от того отключены уже сеансы или нет.

    Кнопка "Блокировка" открывает форму "Блокировка  соединения". С помощью данной формы можно установить блокировку установки соединений с ИБ. При этом запрещено подключение к ИБ новых сеансов, уже существующие сеансы будут отключены в течение времени указанного в поле "Параметр" данной формы (формате отключения в сек.). Можно задать время начала и окончания блокировки соединений. При установке блокировки все сеансам, включая текущий, будет отправлено предупреждение об отключении. Таймаут отключения делится на 5 равных временных интервалов. По истечении 2-х интервалов будет предложено завершить работу с сохранением всех редактируемых данных, 3-х - сеанс будет завершен принудительно, 5-ти сеанс будет завершен без сохранения редактируемых данных и не смотря на открытые модальные окна (аварийно). В случае, если время начала блокировки не указано, от подсчет предупреждений произойдет по схеме отключения с предупреждением. По завершении сеансов 1С:Предприятия будут выполнятся их повторные запуски вплоть до снятия блокировки. Блокировка и отключение устанавливается для всех соединений, включая текущее. Для снятия блокировки необходимо зайти в ИБ с ключом разрешения блокировки, открыть данную форму и нажать "Снять".
