Обработка "АРМ Корзина"

Назначение

Обработка предназначена для выполнения операций с корзиной товаров, подобранных для конкретного клиента. Корзина может быть сохранена, перенесена в документы для оформления покупки, заказа либо выполнения каких-либо других действий.

Относится к подсистеме: «Автоматизированные рабочие места».

Формы

Интерфейс "Корзина"

Является основной формой АРМ. Форма состоит из следующих элементов:

  • Панель "Заголовок"

    • Меню основных действий. В заголовке меню отображается номер текущей корзины. Меню содержит следующие действия:
      1. Записать. Сохраняет корзину в регистр.
      2. Загрузить. Загружает сохраненную ранее корзину из регистра.
      3. Очистить контекст. Создает новую корзину и загружает её в АРМ в качестве текущей. При этом все параметры сбрасываются по умолчанию.
    • Комментарий. Произвольный комментарий к корзине.

  • Область "Параметры расчета"

      • Цены и валюта. Кнопка перехода к форме "Цены и валюта". Форма используется, когда необходимо изменить для корзины следующие реквизиты: валюту, тип цены, скидку или вид оплаты.
      • Клиент Обязательный для заполнения. Ссылается на справочник «Контрагенты и контакты» (может быть задан строкой). Клиент, для которого формируется корзина.
      • Договор. Ссылается на справочник «Договоры взаиморасчетов». Договор, по которому ведутся взаиморасчеты с клиентом.
      • Тип цен. Обязательный для заполнения. Ссылается на справочник «Типы цен». Тип цен, в котором указаны цены на товары.
      • Маркетинговая программа. Ссылается на справочник «Маркетинговые программы». Маркетинговая программа, в рамках которой устанавливается скидка на товары.
      • Скидка/наценка. Ссылается на справочник «Типы скидок и наценок». Скидка/наценка на товары в корзине.
      • Склад. Ссылается на справочник «Склады (места хранения) компании». Склад компании, где хранятся товары корзины.
      • Источник запроса. Обязательный для заполнения. Ссылается на справочник «Источник запроса». Источник запроса (цель подбора). Используется для регистрации упущенного спроса. 
      • Телефон. Телефон клиента. Может быть выбран из списка, отредактирован или введен новый. Редактируется в диалоге. Для нового номера по умолчанию подставляется вид контактной информации "мобильный телефон". Если телефон с данным видом контактной информации уже задан в регистре для данного контрагента, то задается вопрос о замене телефона либо выборе другого вида контактной информации для записи.

  • Корзина

    • Наличие замен. В колонке отображается иконка "Замены" в случае, когда данная номенклатура имеет замены или является заменой другой номенклатуры. По нажатию на колонку открывается обработка «Подбор замен».
    • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Номенклатура (товар, услуга и т.п.).
    • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество единиц измерения номенклатуры.
    • Единица измерения. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Единицы измерения». Единица измерения номенклатурной позиции.
    • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент пересчета по отношению к базовой единице номенклатуры.
    • Количество базовое. Тип значения: «Число». Количество номенклатуры в базовых единицах (коэффициент=1).
    • Цена. Тип значения: «Число». Цена номенклатурной позиции.
    • Сумма. Тип значения: «Число». Сумма всегда без скидки. НДС входит в сумму если это указано в типе цен. Сумма всегда равна Цена*Количество.
    • Процент скидки. Тип значения: «Число». Процент скидки на отгружаемый товар.
    • Скидка. Тип значения: «Число». Сумма скидки по строке документа.
    • Всего. Тип значения: «Число». Итоговая сумма по строке документа со всеми налогами.
    • Процент НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по номенклатурной позиции.
    • НДС. Тип значения: «Число». Сумма НДС по номенклатурной позиции.
    • Характеристика номенклатуры. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры, например, цвет, размер и т.п.
    • Партия. Ссылается на план видов характеристик «Типы партий». Партия, с которой списывается товар.
    • ГТД. Ссылается на справочник «ГТД». Номер грузовой таможенной декларации (для товаров, поступивших по импорту).
    • Ячейка. Ссылается на справочник «Ячейки хранения». Ячейка хранения для номенклатуры. Значение по умолчанию хранится в номенклатуре в разрезе складов.

  • Панель информации

    Состоит из закладок, на которых отображается различная информация о номенклатуре. Информация формируется при активизации  какой-либо строки в таблице товаров.  В командных панелях на закладках имеется подменю со списком отчетов. При открытии отчеты формируются с фильтром по номенклатуре в корзине.

    Для некоторых закладок имеется режим "Только остатки". Если режим включен, то для строк показываются колонки с остатками количества (суммы) по регистрам остатков. Если режим выключен, то для строк дополнительно показываются колонки с количеством (суммой) из документов независимо от остатков на текущий момент.  

      • 1 Остатки

      Содержит информацию об остатках номенклатурной позиции на складах компании. В шапке содержится надпись "Минимальный статок" - минимальный остаток товара, необходимый для обеспечения потребностей текущего подразделения.

      • Склад компании. Ссылается на справочник «Склады компании». Склад компании, на который были поступления номенклатуры.
      • Характеристика. Ссылается на справочник «Характеристики номенклатуры». Характеристика номенклатуры, для которой отображаются показатели.
      • Остаток на складе. Тип значения: «Число». Количество товара в наличии на указанном складе.
      • Из них в резерве. Тип значения: «Число». Количество зарезервированного товара из оставшегося на указанном складе.
      • Свободный остаток. Тип значения: «Число». Количество товара в свободном остатке на указанном складе.
      • Ожидается. Тип значения: «Число». Количество товара, которое ожидается к поступлению по непроведенным документам "Поступление товаров".
      • Цена розничная. Тип значения: «Число». Цена по данным складского учета (регистр "Остатки товаров компании"). Вычисляется как сумма, поделенная на количество.
      • Сумма розничная. Тип значения: «Число». Сумма по данным складского учета (регистр "Остатки товаров компании").

      2 В производстве

      Содержит сведения о заказ-нарядах, в которых данная номенклатурная позиция в настоящий момент находится в производстве.

      • Заказ-наряд. Тип значения: «ДокументСсылка.ЗаказНаряд». Документ, согласно которому номенклатура используется в производстве.
      • Цех. Ссылается на справочник «Цеха». Цех, в котором выполняются работы по указанному заказ-наряду.
      • Количество (всего). Тип значения: «Число». Количество номенклатуры в производстве по указанному заказ-наряду (количество, указанное в документе).
      • Количество (остаток). Тип значения: «Число». Количество номенклатуры в производстве по указанному заказ-наряду (остаток на текущую дату).

      3 Партии

      Содержит информацию по движениям партий данной номенклатурной позиции.

      • Партия. Ссылается на документы, по которым были движения номенклатуры.
      • Характеристика. Ссылается на справочник «Характеристики номенклатуры». Характеристика номенклатуры, для которой отображаются показатели.
      • Контрагент. Ссылается на справочник «Контрагенты и контакты». Контрагент, указанный в документе, по которому были движения номенклатуры.
      • Количество (всего). Тип значения: «Число». Остаток номенклатуры на складе (количество, указанное в документе).
      • Себестоимость (всего). Тип значения: «Число». Себестоимость номенклатуры в валюте управленческого учета (сумма, указанная в документе).
      • Цена.Тип значения: «Число». Цена номенклатуры в валюте управленческого учета.
      • Количество (остаток). Тип значения: «Число». Остаток номенклатуры на складе (остаток на текущую дату).
      • Себестоимость (остаток).Тип значения: «Число». Себестоимость номенклатуры в валюте управленческого учета (цена*остаток на текущую дату).

      4 Заказы покупателей

      Содержит информацию по движениям заказов покупателей данной номенклатурной позиции. На панели расположены кнопки для выполнения различных операций с заказами покупателей. Операции выполняются с заказом в текущей строке и для номенклатуры в текущей строке корзины. Подробнее операции описаны в справочной информации к «Форме операций с заказами покупателей».

      • Заказ. Ссылается на документы, по которым были движения заказов номенклатуры.
      • Заказчик. Ссылается на справочник «Контрагенты и контакты». Контрагент, указанный в заказе.
      • Документ-основание. Ссылается на документ, на основании которого был введен заказ.
      • Вид ремонта. Ссылается на справочник «Виды ремонта». Ссылается на вид ремонта заказ-наряда, на основании которого введен заказ покупателя.
      • Тип заказа. Ссылается на справочник «ТипыЗаказов». Один из возможных типов заказов.
      • Заказано (всего). Тип значения: «Число». Заказанное количество товара (количество, указанное в документе).
      • Заказано (остаток). Тип значения: «Число». Остаток заказанного количества товара (остаток на текущую дату).
      • Из них в резерве. Тип значения: «Число». Остаток зарезервированного количества товара по заказу (остаток на текущую дату).
      • Распределено. Тип значения: «Число». Количество товара по заказу, распределенное по заказам поставщиков (остаток на текущую дату).

      5 Заказы поставщикам

      Содержит информацию по движениям заказов поставщикам данной номенклатурной позиции.

      • Заказ. Ссылается на документы, по которым были движения заказов номенклатуры.
      • Поставщик. Ссылается на справочник «Контрагенты и контакты». Контрагент, указанный в заказе.
      • Тип заказа. Ссылается на справочник «ТипыЗаказов». Один из возможных типов заказов.
      • Заказано (всего). Тип значения: «Число». Заказанное количества товара (количество, указанное в документе).
      • Заказано (остаток). Тип значения: «Число». Заказанное количества товара (остаток на текущую дату).
      • Остаток распределения. Тип значения: «Число». Остаток заказанного количества товара, которое не распределено по заказам покупателей. 

      6 Остатки по ячейкам

      Содержит информацию об остатках данной номенклатурной позиции в ячейках на складах компании. В шапке содержится надпись "Ячейка хранения по умолчанию" - это ячейка, которая используются для автоподстановки в документы для данной номенклатурной позиции в качестве ячейки хранения по умолчанию. Задается в карточке номенклатуры для каждого склада.

      • Склад компании. Ссылается на справочник «Склады компании». Склад компании,в котором размещается номенклатура .
      • Ячейка. Ссылается на справочник «Ячейки хранения». Ячейка указанного склада, в которой размещается номенклатура.
      • Остаток по ячейке. Тип значения: «Число». Общее количество товара в указанной ячейке на указанном складе.
      • Остаток по складу. Тип значения: «Число». Общее количество товара на указанном складе.

      7 Цены

      Содержит информацию о ценах на данную номенклатурную позицию. Включает цены контрагентов и рассчетные цены.

      • Тип цен. Ссылается на справочник «Типы цен». Тип цены, который установлен для номенклатуры.
      • Контрагент. Ссылается на справочник «Контрагенты и контакты». Контрагент, для которого установлена цена.
      • Валюта. Ссылается на справочник «Валюты». Валюта, в которой указана цена номенклатуры.
      • Характеристика. Ссылается на справочник «Характеристики номенклатуры». Характеристика номенклатуры, для которой отображаются показатели.
      • Единица измерения. Ссылается на справочник «Единицы измерения». Единица измерения номенклатуры.
      • Подразделение компании. Ссылается на справочник «Подразделения компании». Подразделение компании, где используется указанная цена на номенклатуру.
      • Цена. Тип значения: «Число». Числовое выражения цены номенклатуры.

      8 Продажи

      Содержит отчет, отображающий сведения о финансовых и количественных показателях продаж по данной номенклатурной позиции за выбранный период. Отчет может быть визуально представлен виде табличного документа или диаграммы. Для отчета доступна настройка показателей, группировок, интервала, а также стиля (вида диаграммы).

      9 Прайс-листы

      Содержит список номенклатуры в прайс-листах контрагентов с таким же артикулом как и у данной номенклатурной позиции, как загруженных в информационную базу данных, так и находящихся во внешних источниках данных.

      • Прайс-лист. Ссылается на справочник «Прайс-листы контрагентов». Наименование прайс-листа, где содержится номенклатура.
      • Номенклатура. Ссылается на справочник «Номенклатура». Номенклатура в прайс-листе.
      • № по каталогу. Тип значения: «Строка». Артикул номенклатуры в прайс-листе.
      • Производитель. Ссылается на справочник «Контрагенты». Производитель номенклатуры в прайс-листе.
      • Наименование. Тип значения: «Строка». Наименование номенклатуры в прайс-листе.
      • Цена. Тип значения: «Число». Цена номенклатуры в прайс-листе.
      • Количество. Тип значения: «Число». Количество номенклатуры в прайс-листе.
      • Срок поставки. Тип значения: «Число». Срок поставки номенклатуры.

      10 Аналоги

      Содержит список "заменителей" данной номенклатурной позиции.

      • № по каталогу. Тип значения: «Строка». Артикул аналога номенклатуры.
      • Производитель. Ссылается на справочник «Контрагенты». Производитель указанного аналога номенклатуры.
      • Номенклатура. Ссылается на справочник «Номенклатура». Номенклатура - аналог.
      • Главный по группе. Тип значения: «Булево». Показывает, что указанный аналог является главным в группе аналогов.
      • Главный по производителю. Тип значения: «Булево». Показывает, что указанный аналог является главным в группе аналогов по производителю.
      • Остаток. Тип значения: «Число». Остаток аналога номенклатуры по всем складам компании в целом.

     

    • Панель основных действий

    Содержит кнопки основных действий АРМ:

      • Печать. Печать корзины. Печать этикеток и ценников для товаров в корзине, печать списка элементов корзины.
      • Из файла. Открытие обработки "Загрузка данных из табличного документа" для загрузки/выгурзки товаров корзины.
      • Из документа. Загружает корзину из табличной части выбранного документа.
      • В документ. Выгружает корзину в табличную часть выбранного документа.
      • Создать. Выгружает корзину в табличную часть нового документа выбранного вида.
      • Сохранить. Записывает корзину в регистр «Корзина».
      • Загрузить. Загружает сохраненную ранее корзину из регистра «Корзина».
      • Закрыть. Закрывает форму.

    Способ вызова

    Открывается из обработки "Функциональная панель". Отдельно не используется.
