Обработка "АРМ Клиенты и автомобили"

Назначение

Обработка представляет собой рабочее место мастера-консультанта и предназначена для идентификации клиента и автомобиля с целью выполнения дальнейших действий.

Относится к подсистеме: «Автоматизированные рабочие места».

Формы

Интерфейс "Клиенты и автомобили"

Является основной формой АРМ. Форма состоит из следующих элементов:

  • Панель "Заголовок"

    • Строка поиска. Поиск осуществляется по справочникам "Контрагенты" и "Автомобили" по следующим полям: наименование клиента, контактная информация, государственный номер автомобиля, VIN-код автомобиля, номер дисконтной карты. Поиск выполняется при изменении значения в строке поиска по нажатии на клавишу выбора "Enter". Найденные клиенты будут отображены в списке клиентов. Если найдены автомобили, то они будут отображены в списке автомобилей, а их владельцы - в списке клиентов.  Предусмотрено хранение истории поиска, которая доступна в выпадающем списке. История хранится пока запущена форма-владелец - функциональная панель.
    • Флаги управления отбором. Используются для наложения дополнительных отборов на список клиентов. Дополнительные отборы применяются и для списка, полученного в результате поиска.
    • "Только мои клиенты"  - отображаются только клиенты, для которых имеются заявки на ремонт, автором которых является текущий пользователь.
    • "Записанные на сегодня" - отображаются только клиенты, для которых имеются заявки на ремонт с работами, запланированными на сегодняшний день.
    • "Ожидающие выдачи" - отображаются только клиенты, для которых имеются заказ-наряды со статусом "выполнен". 
    • Кнопка  "Отключить отбор". Используется для отключения отбора в списке, полученного в результате поиска или наложения фильтров.
    • Кнопка "Настройка полей поиска". Используется для настройки списка полей, по которым производится поиск. В форме настройки полей поиска обязательно должно быть выбрано хотя бы одно поле.

 

  • Область клиентов

    • Список "Клиент". Служит для поиска и идентификации клиента. Имеется командная панель с возможностью добавления и редактирования записей. По умолчанию содержит полный список контрагентов. На список могут быть наложены фильтры в результате поиска или установки флагов.
    • Поле "Информация о клиенте". Содержит сведения о выбранном в списке клиенте.
    • Поле "Контактная информация". Показывает контактную информацию о выбранном в списке клиенте, а также других контрагентах, контактным лицом которых он является.

 

  • Область автомобилей

    • Список "Автомобиль". Служит для поиска и идентификации автомобиля клиента. В списке черным цветом отображаются автомобили, владельцем которых в настоящий момент является выбранный клиент, серым - автомобили, владельцем которых он был ранее. Имеется командная панель с возможностью добавления и редактирования записей. При добавлении нового автомобиля из АРМ выбранный клиент подставляется в качестве владельца автомобиля.
    • Поле "Информация об автомобиле". Содержит сведения о выбранном в списке автомобиле.
    • Поле "История автомобиля". Показывает список документов "Заказ-наряд" и "Заявка на ремонт" с отбором по выбранному в списке автомобилю.

 

  • Панель основных действий

Содержит кнопки основных действия и контекстных переходов между АРМ.

Контекстные переходы к другим АРМ:

    • «Калькуляция»
      1. АРМ "Калькуляция" не запущен или запущен и открыта новая несохраненная заявка на ремонт - выполняется подстановка клиента (в качестве заказчика и плательщика) и автомобиля.
      2. АРМ "Калькуляция" запущен и открыта записанная заявка на ремонт - создаётся новая заявка на ремонт, выполняется подстановка клиента (в качестве заказчика и плательщика) и автомобиля.
    • «Запись на ремонт»
      1. АРМ "Запись на ремонт" не запущен или запущен и открыта новая несохраненная заявка на ремонт - выполняется подстановка клиента (в качестве заказчика и плательщика) и автомобиля.
      2. АРМ "Запись на ремонт" запущен и открыта записанная заявка на ремонт - создаётся новая заявка на ремонт, выполняется подстановка клиента (в качестве заказчика и плательщика) и автомобиля.
    • «Машинозаезд»
      1. Выбран только клиент (автомобилей в списке нет) - создаётся новый заказ-наряд без автомобиля. Клиент подставляется в качестве заказчика и плательщика по заказ-наряду.
      2. Для выбранного клиента и автомобиля имеются открытые машинозаезды (хотя бы один заказ-наряд имеет статус, отличающийся от "закрыт") - открывается последний открытый машинозаезд.
      3. Для выбранного клиента и автомобиля открытых машинозаездов нет - создаётся новый заказ-наряд с датой машинозаезда, равной текущей дате. В созданный заказ-наряд выполняется подстановка клиента (в качестве заказчика и плательщика) и автомобиля.

Основные действия:
    • Звонок клиенту  - переход к обработке "СофтФон".
    • SMS клиенту - переход к созданию нового SMS сообщения
    • E-mail клиенту - переход к созданию нового электронного письма
    • Закрыть - закрыть форму

Способ вызова

Открывается из обработки "Функциональная панель". Отдельно не используется.
