Обработки "Обмен с удаленными подразделениями"

Назначение

Обработка "Обмен с удаленными подразделениями" предназначена для автоматического формирования и последующей публикации, и/или получения сообщений обмена. В качестве узла обмена может быть указана ссылка на узел плана обмена "Удаленные подразделения". В этом случае, возможно выполнение одиночных операций публикации/получения сообщений обмена, таких как:
 - Выполнение загрузки сообщения обмена (гиперссылка "Выполнить загрузку")
 - Выполнение выгрузки сообщения обмена (гиперссылка "Выполнить выгрузку")
 - Выполнение последовательной операции загрузки/выгрузки сообщений обмена (гиперссылка "Выполнить обмен")

В случае, если узел плана обмена не указан, то будет доступна только одна операция - получение и отправка сообщений обмена для всех узлов плана обмена "Удаленные подразделения" (гиперссылка "Выполнить обмен со всеми узлами").

Относится к подсистеме: «Сервис».

Реквизиты

  • Узел обмена. Ссылается на план обмена «Удаленные подразделения». Узел, с которым производится обмен.
  • Настройка обмена. Ссылается на справочник «Настройки доставки сообщений». Указывает способ доставки сообщения.
  • Комментировать. Тип значения: «Булево». Признак необходимости комментирования процесса обмена.
  • Комментировать объекты. Тип значения: «Булево». Признак необходимости комментирования процесса обмена по отдельным объектам. 
  • Допроводить по партиям. Тип значения: «Булево». Признак необходимости допроведения по партиям по окончании обмена.
  • Количество элементов в транзакции. Тип значения: «Число». Задает количество элементов данных в одной транзакции при запуске сеанса обмена.

Способ вызова

Вызывается из формы списка плана обмена «Удаленные подразделения».

Дополнительная информация

Если в получаемом сообщении обмена содержится информация об изменении конфигурации, то будет выдано соответствующий вопрос об обновлении конфигурации через определенное время, отведенное пользователям для завершение работы, с последующим перезапуском текущего сеанса. При отказе от обновления конфигурации необходимо самостоятельно зайти в Конфигуратор и обновить конфигурацию базы данных, после чего продолжить обмен. При запуске обработки из другого контекста вопроса не будет и обновление конфигурации и конфигурации произойдет автоматически, после чего также автоматически продолжиться обмен.
