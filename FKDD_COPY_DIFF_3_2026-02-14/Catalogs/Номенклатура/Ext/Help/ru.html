Справочник "Номенклатура"

Назначение

В справочник "Номенклатура" заносится информация обо всех товарах и услугах, включаемых в оформляемые в компании документы, а также информация о таре, наборах товаров и комплектах.

Относится к подсистеме: «Номенклатурные».

Предопределенные элементы

  • Автомобиль. Используется для автоподстановки в некоторых объектах конфигурации.
  • Авторабота. Используется для автоподстановки в некоторых объектах конфигурации.
  • Комиссионное вознаграждение. Используется для автоподстановки при оформлении комиссионного вознаграждения.
  • Опция производителя. Используется для автоподстановки в некоторых объектах конфигурации. 
  • Предоплата. Используется для автоподстановки в документ Счет-фактура выданный, вводимый на основании ПКО.

Параметры

Длина кода: 8, тип кода: Строка.

Длина наименования: 80.

Иерархический.

Реквизиты

  • Иностранное наименование. Тип значения: «Строка». Наименование на иностранном языке. Например, из зарубежных каталогов.
  • № по каталогу. Тип значения: «Строка». Артикул - код производителя.
  • Полное наименование. Тип значения: «Строка». Полное наименование. Например, для печати.
  • Тип номенклатуры. Обязательный для заполнения. Ссылается на справочник «Типы номенклатуры». Это основной признак номенклатурной позиции, определяющий её назначение и поведение в конфигурации, способ формирования и декодирования штрихкода.
  • Вид номенклатуры. Обязательный для заполнения. Ссылается на перечисление «Виды номенклатуры». Вид номенклатуры. Может принимать значения: товар, услуга, комплект, тара, набор и другие.
  • Базовая единица измерения. Обязательный для заполнения. Ссылается на справочник «Классификатор единиц измерения». Базовая единица измерения номенклатуры.
  • Основная единица измерения. Обязательный для заполнения. Ссылается на справочник «Единицы измерения». Основная единица измерения номенклатуры. При создании нового элемента справочника автоматически создается основная единица измерения, на основании базовой единицы измерения.
  • Ставка НДС. Обязательный для заполнения. Ссылается на справочник «Ставки НДС». Ставка НДС по умолчанию для оформления документов.
  • Процент наценки. Тип значения: «Число». Процент наценки по умолчанию (для расчета цен). При создании нового элемента номенклатуры % наценки по умолчанию берется из группы номенклатуры. % наценки в группе номенклатуры влияет только на заполнение % наценки для новых элементов и не влияет на подстановку % в документы.
  • Валюта учета. Ссылается на справочник «Валюты». Используется для ведения дополнительного учета валютной себестоимости и цен товара.
  • Страна происхождения. Ссылается на справочник «Классификатор стран мира». Страна происхождения по умолчанию.
  • Комментарий. Тип значения: «Строка». Любая дополнительная информация.
  • Способ распределения на себестоимость прихода. Заполняется для услуг. Ссылается на перечисление "Способы распределения доп. расходов". Управляет распределением суммы по услуге в пределах документа.
  • Статья доп расходов. Заполняется для услуг. Ссылается на справочник «Статьи доходов и расходов». По этой статье происходит начисление расходов / доходов при поставке / отгрузке.
  • Вид платежа. Заполняется для услуг. Тип значения: «Строка». Вид платежа для оплаты услуг связи через электронную систему CyberPlat. Наименование вида платежа должно либо полностью соответствовать названию оператора (элементу справочника «Виды платежей»), либо частично повторять его первые символы.
  • Счет учета НДС по реализации. Заполняется для услуг. Ссылается на план счетов "Регламентированный". Счет учета НДС по реализации. Используется при выгрузке в бухгалтерию.
  • Запрет продажи. Тип значения: «Булево». Признак запрета реализации номенклатуры.
  • Запрет закупки. Тип значения: «Булево». Признак запрета закупки номенклатуры.
  • Снята с производства. Тип значения: «Булево». Признак снятия с производства.
  • Кратность поставок. Тип значения: «Число». Кол-во которому бедет кратно количество в заказе поставщику.
  • Вес. Тип значения: «Число». Обычно соответствует массе Брутто в основной единице измерения массы.
  • Объем. Тип значения: «Число». Обычно соответствует объему в основной единице измерения объема. 
  • Прослеживаемый товар. Тип значения: «Булево». Признак того, что данный товар является прослеживаемым.

13 Замены

Старые замены. Список артикулов номенклатуры, для которых данная номенклатура является заменой.

  • Шаг. Тип значения: «Число». Шаг равен "-1" если данная номенклатура является непосредственной заменой, которая задана в регистре «Замены». Если данная номенклатура является заменой через 1 другую номенклатуру, то шаг равен "-2", и так далее.
  • № по каталогу. Тип значения: «Строка». Каталожный номер номенклатуры, для которой данная является заменой.
  • Производитель. Ссылается на справочник «Пользователи». Производитель номенклатуры, указанной в строке.
  • Номенклатура. Ссылается на справочник «Номенклатура». Номенклатура, для которой данная является заменой.

Новые замены. Список артикулов номенклатуры, являющихся заменой для данной номенклатурной позиции.

  • Шаг. Тип значения: «Число». Шаг равен "1" если данная номенклатура имеет непосредственную замену, которая задана в регистре «Замены». Если данная номенклатура имеет замены через 1 другую номенклатуру, то шаг равен "2", и так далее.
  • № по каталогу. Тип значения: «Строка». Каталожный номер номенклатуры, которая является заменой для данной.
  • Производитель. Ссылается на справочник «Пользователи». Производитель номенклатуры, указанной в строке.
  • Номенклатура. Ссылается на справочник «Номенклатура». Номенклатура, которая является заменой для данной. 

14 Связанная номенклатура

Содержит список связанной номенклатуры данной номенклатурной позиции.

  • Связанная номенклатура. Ссылается на справочник «Номенклатура». Связанная номенклатура.
  • Модель. Ссылается на справочник «Модели». Модель автомобиля, для которой предназначена связанная номенклатура.
  • Вариант комплектации. Ссылается на справочник «Варианты комплектации». В соотвествии с выбраной моделью можно указать вариант комплектации, для которой можно указать связанную номенклатуру.
  • Количество. Тип значения: «Число». Требуемое количество связанной номенклатуры.

15 Применяемость

Содержит список моделей автомобилей, для которых применяется данная номенклатурная позиция.

  • Модель. Ссылается на справочник «Модели автомобилей». Модель автомобиля, для которой применяется номенклатура.

16 Минимальные остатки

Содержит сведения о минимальном остатке номенклатуры, которое необходимо для обеспечения потребностей подразделений.

  • Период. Тип значения: «Дата». Дата, начиная с которого действует запись.
  • Подразделение компании. Ссылается на справочник «Подразделения». Подразделение, для которого указывается минимальный остаток.
  • Остаток. Тип значения: «Число». Минимальное количество номенклатуры для указанного подразделения.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов.
  • Форма группы. Основная форма группы. Используется для заполнения обязательных реквизитов группы.
  • Форма свойств. Форма свойств справочника. Используется для заполнения необходимых свойств объекта.
  • Форма выбора группы. Основная форма выбора группы. Используется для выбора групп.

Способ вызова

Вызывается из меню "Справочники --> Номенклатурные --> Номенклатура".
