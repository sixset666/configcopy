Справочник "Контрагенты и контакты"

Назначение

Контрагент – это общее понятие, куда включены поставщики и покупатели, организации и частные лица. Справочник хранит список контрагентов и их контактных лиц, т.е. юридических или физических лиц внешних или внутренних по отношению к нашей компании, с которыми связана непосредственная деятельность компании, и на/от которых выписываются первичные документы. Контрагенты могут быть разных видов, например, покупатель, поставщик, подотчетное лицо и т.д.

Контрагенты могут быть сгруппированы по группам классификатора справочника, например, «Поставщики», «Покупатели» и «Реализаторы». Справочник контрагентов имеет неограниченное количество уровней вложенности. Информация о контрагентах располагается на нескольких закладках. Любой из контрагентов может являтся контактным лицом другого. Для контрагентов с видом контактное лицо запрещено вводить документы.

Относится к подсистеме: «Идентификационные».

Предопределенные элементы

  • Основной покупатель. Покупатель по умолчанию. Элемент можно изменить. 
  • Основной поставщик. Поставщик по умолчанию. Элемент можно изменить.

Параметры

Длина кода: 8, тип кода: Строка.

Длина наименования: 50.

Иерархический.

Реквизиты

  • Полное наименование. Тип значения: «Строка». Реквизит является обязательным для заполнения. Полное наименование контрагента.
  • Тип клиента. Реквизит является обязательным для заполнения. Ссылается на перечисление «Формы собственности». Форма собственности контрагента. Может принимать следующие значения: юридическое лицо, частный предприниматель, частное лицо, прочее.
  • Вид отношений. Реквизит является обязательным для заполнения. Ссылается на перечисление «Виды контрагентов». Характеризует вариант взаимоотношений с контрагентом. Возможные значения: покупатель, поставщик, подотчетное лицо, филиал, собственник, прочее.
  • Фамилия. Тип значения: «Строка». Реквизит доступен при выбораном типе клиента "Частное лицо".Реквизит является обязательным для заполнения.
  • Имя. Тип значения: «Строка». Реквизит доступен при выбораном типе клиента "Частное лицо". Реквизит является обязательным для заполнения.
  • Отчетсво. Тип значения: «Строка». Реквизит доступен при выбораном типе клиента "Частное лицо". Реквизит является обязательным для заполнения.  
  • Срок поставки. Тип значения: «Число». Реквизит доступен при выбораном виде отношений "Поставщик". Количество дней, в течение которых обычно осуществляется поставка данным поставщиком.
  • Сотрудник. Ссылается на справочник «Сотрудники». Реквизит доступен при выбранном виде отношений "Подотчетное лицо". В этом случае является обязательным для заполнения. Устанавливает связь между контрагентом подотчетным лицом и сотрудником компании ).
  • Филиал. Ссылается на справочник «Организации». Реквизит доступен при выбранном виде отношений "Филиал". В этом случае является обязательным для заполнения. Устанавливает связь между контрагентом - филиалом компании и организацией компании).
  • Кураторы. Ссылается на регистр сведений «Кураторы». Список сотрудников компании отвечающих за работу с контрагентом. В регистре указывается сотрудник компании и «договор» с контрагентом.
  • Комментарий. Тип значения: «Строка». Дополнительная информация о контрагенте.
  • ИНН. Тип значения: «Строка». Идентификационный номер налогоплательщика. Не активно при типе клиента "Частное лицо".
  • Головной контрагент. Тип значения: «Контрагенты и контакты». Юридическое лицо, информация которого использульзуется для данного контрагента. Доступен при типе клиента "Обособленное подразделение".
  • Код по ОКПО. Тип значения: «Строка». Код контрагента по Общероссийскому Классификатору Предприятий и Организаций. Не доступно при типе клиента "Частное лицо".
  • КПП. Тип значения: «Строка». Код причины постановки на учет. Формируется из ИНН. Не доступно при типе клиента "Частное лицо".
  • Код ИМНС. Тип значения: «Строка». Код налоговой инспекции контрагента. 4 первых цифры ИНН. Не доступно при типе клиента "Частное лицо".
  • Дата рождения. Тип значения «Дата». Активно при типе клиента "Частное лицо". Дата рождения контрагента.
  • Пол. Тип значения «Строка». Доступно при типе клиента "Частное лицо". Пол контрагента.
  • Источники информации. Ссылается на справочник «Источники информации». Внешние источники информации о контрагенте.
  • Не является плательщиком НДС. Если флажок установлен, то при заполнении документов контрагента НДС учитываться не будет.
  • График работы. Ссылается на справочник «Графики работы». График работы контрагента.
  • Основной банковский счет. Ссылается на справочник «Банковские счета». Банковский счет контрагента, используемый по умолчанию. 
  • Выгружать автомобили и заказ-наряды на сайт. Тип значения: «Булево». При установке признака в личный кабинет клиента на сайте будут выгружаться сведения обо всех автомобилях и заказ-нарядах.
  • Согласие на получение SMS. Тип значения: «Булево». При согласии на обработку персональных данных указывает согласие контрагента на получение SMS.
  • Страна регистрации. Ссылается на справочник «Классификатор стран мира». Страна контрагента, где он зарегистрирован.
  • Налоговый номер. Тип значения: «Строка». Код налогоплательщика, присвоенный в стране регистрации. Заполняется только для контрагентов не из России.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов. В нижней части формы закладки предназначены для быстрого просмотра информации выделенного контрагента или контакта.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов. Также форма элемента используется для заполнения параметров контрагента или контакта. Форма содержит несколько закладок, каждая из которых устанавливает соответствующие параметры контрагента или контакта.
    • Закладка "Основные". Устанавливаются основные данные о предприятии: тип клиента, вид отношений. Закладка "Контактная информация" предоставляет возможность просмотра и редактирования регистра сведений «Контактная информация» отфильтрованного по контрагенту. Закладка "Дополнительная информация" позволяет заполнить коды регистрации, графики работы, источники информации и признак того, является ли контрагент плательщиком НДС. Закладка "Контакты" предназначена для редактирования списка контактных лиц контрагента или контакта и их контактной информации.  
    • Закладка "Счета и договоры". Предоставляет возможность просмотра и редактирования элементов справочников «Банковские счета» и «Договоры взаиморасчетов» связанных с взаимоотношениями с контрагентом. На закладке также указывается основной банковский счет контрагента.
    • Закладка "Подтверждающие документы". Предоставляет возможность просмотра и редактирования элементов справочника «Подтверждающие документы» связанных с контрагентом.
    • Закладка "Дисконт. карты". Таблица на закладке содержит список дисконтных карт, зарегистрированных на контрагента, с накопленной на них суммой. 
    • Закладка "Документы". Таблица на закладке содержит список документов, сформированных по данному контрагенту. Если данный контрагент имеет вид отношений "контактное лицо", то в таблице также отображаются документы для ближайшего ведущего контрагента. Если данный контрагент не является контактным лицом, то в таблице также отображаются документы других контрагентов, для которых данный контрагент задан ведущим.
  • Форма группы. Основная форма группы. Используется для заполнения реквизитов группы контрагентов, которые будут по умолчанию подставляться при вводе нового контрагента внутри данной группы.  
  • Форма быстрого ввода. Форма предназначена для быстрого ввода контрагента. Вызывается при нажатии на кнопку "быстрый ввод" на форме списка. Форма содержит все необходимые реквизиты для быстрого заполнения информации о контрагенте и о его контактном лице.
  • Форма выбора группы. Форма выбора группы. Вызывается при выборе родителя в форме быстрого ввода. Форма содержит список групп контрагентов.

Способ вызова

Вызывается из меню "Справочники --> Идентификационные --> Контрагенты и контакты".

Дополнительная информация

Во время работы с формой элемента доступны следующие сочетания клавиш для быстрого ввода:

  • [Ctrl+T] - Добавить новый телефон;
  • [Ctrl+W] - Добавить новый адрес Web-страницы;
  • [Ctrl+E] - Добавить новый адрес электронной почты;
  • [Ctrl+R] - Добавить новый адрес.

Если выбрана закладка "Контактная информация" и курсор находится на одной из строк списка контактных лиц, то при нажатии перечисленных выше сочетаний клавиш добавляется контактная информация для выбранного контактного лица, иначе добавляется контактная информация для контрагента.

При работе с формой быстрого ввода доступны следующие сочетания клавиш:

  • [Ctrl+1] - Переход на закладку ввода информации о контрагенте;
  • [Ctrl+2] - Переход на закладку ввода информации о контактном лице контрагента.
