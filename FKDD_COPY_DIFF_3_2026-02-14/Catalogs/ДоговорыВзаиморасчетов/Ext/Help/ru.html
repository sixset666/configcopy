Справочник "Договоры взаиморасчетов"

Назначение

Содержит перечень заключенных договоров с контрагентами. Для каждого контрагента может быть оформлено сколько угодно договоров. Удобство в ведении договоров состоит в том, что все проведенные операции с контрагентом можно разнести по разным договорам, в зависимости от смысла конкретной операции.

Например, можно отнести к одному договору операции приема товаров на комиссию, а к другому — покупки товаров. Тогда в каждом договоре можно увидеть не общие итоги по расчетам с данным контрагентом, а текущее состояние каждой конкретной операции.

Договор определяет валюту взаиморасчетов по данному договору и в случае ведения взаиморасчетов с одним контрагентом в разных валютах необходимо ввести соответственное количество договоров.

Относится к подсистеме: «Идентификационные».

Параметры

Длина кода: 8, тип кода: Строка.

Длина наименования: 70.

Не иерархический.

Владельцы справочника: «Контрагенты и контакты».

Реквизиты

  • Вид договора. Реквизит является обязательным для заполнения. Ссылается на перечисление «Виды договоров». Вид договора с контрагентом. Может принимать следующие значения: поставка, продажа, с комиссионером, с комитентом,  прочее, кредит.
  • Тип договора. Реквизит является обязательным для заполнения. Ссылается на перечисление «Типы договоров». Тип договора может принимать следующие значения: Виртуальный договор, Договор, Приложение.
  • Номер договора. Тип значения: «Строка». Номер договора с контрагентом.
  • Дата начала. Тип значения: «Дата». Реквизит является обязательным для заполнения. Дата начала учета по договору.
  • Дата окончания. Тип значения: «Дата». Дата окончания учета по договору.
  • Валюта. Реквизит является обязательным для заполнения. Ссылается на справочник «Валюты». Определяет валюту взаиморасчетов по договору. Не может меняться, если по договору проведен хотя бы один документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Организация нашей компании, от которой ведется договор с контрагентом.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение, заключившее договор с контрагентом.
  • Основной. Тип значения: «Булево». Используется при автоматическом выборе договора. Договор становится более приоритетным перед остальными договорами контрагента в разрезе подразделения и организации.
  • Область деятельности. Флаги отражающие область применения договора. Если установлен флаг "з\ч и сервис", то по договору разрешено оформление документов по запчастям и автосервису. Если установлен флаг "т\с", то разрешено оформление документов по транспортным средствам. Если установлен флаг "Прочее", то разрешено оформление внутренних документов.
  • Кураторы. Ссылается на справочник «Сотрудники». Сотрудник, курирующий указанного контрагента.
  • Тип цен покупки. Ссылается на справочник «Типы цен». Тип цен по умолчанию для оформления покупок по данному договору.
  • Тип цен продажи. Ссылается на справочник «Типы цен». Тип цен по умолчанию для оформления продаж по данному договору.
  • Тип цен работ. Ссылается на справочник «Типы цен». Тип цен по умолчанию для оформления работ заказ-наряда по данному договору.
  • Вид оплаты. Реквизит является обязательным для заполнения. Ссылается на перечисление «Виды оплаты». Вид оплаты по договору. Может принимать следующие значения: наличный расчет, безналичный расчет, произвольная оплата.
  • Тип скидки наценки. Ссылается на справочник «Типы скидок и наценок». Тип скидки наценки по умолчанию для оформления продаж по данному договору.
  • Макс. сумма кредита. Тип значения: «Число». Максимальная сумма кредита (долга) по договору в валюте договора.
  • Отмена контроля суммы кредита. Тип значения: «Булево». Установка контроля проверки превышения максимальной суммы кредита. При установке данного флага контроль проверки не производится. Для нового договора взаиморасчетов по умолчанию взводится данный флаг. 
  • Процент предоплаты. Тип значения: «Число». Процент предоплаты по договору.
  • Срок оплаты задолженности. Тип значения: «Число». Срок оплаты задолженности (в днях).
  • Процент комиссионного вознаграждения. Тип значения: «Число». Процент комиссионного вознаграждения для договоров комиссии. Поле активно при выборе вида договора "Комиссия".
  • Сумма отчета. Тип значения: «Булево». Способ заполнения сумм в отчете комитенту по сумме поступления или по сумме продажи комиссионного товара. ЛОЖЬ='По суммам продажи'; ИСТИНА='По сумме поставки'.
  • Автоматическое закрытие сделок. Тип значения: «Булево». Если флажок установлен, то при вводе документов плат по умолчанию будет установлен флаг, по которому, при проведении документа, сначала будут оплачены все незакрытые сделки.  
  • Способ закрытия сделок. Реквизит является обязательным для заполнения. Ссылается на перечисление «Способ закрытия сделок». Задает способ взаиморасчета по сделкам, которые сами не производят движений по взаиморасчетам с контрагентами. Ссылается на перечисление "Способ закрытия сделок": По расчетным документам - движения по взаиморасчетам будет осуществлено на сделку-основание или на документ, осуществляющий движение. По счетам и заказам - движения по взаиморасчетам будет отнесено на счет или заказ контрагента, если таковые имеются в цепочке подчиненных документов.
  • Статья ДДС. Ссылается на справочник «Статьи ДДС». Статья движения денежных средств.
  • Автор. Ссылается на справочник «Пользователи». Пользователь, создавший договор.
  • Расчеты в условных единицах. Тип значения: «Булево». При установленном флаге оплата по валютному договору производится в условных единицах. 
  • Дата создания. Тип значения: «Дата». Дата и время создания договора. 
  • Комментарий. Тип значения: «Строка». Любая дополнительная информация.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов. При создании нового договора взаиморасчетов необходимо выбрать контрагента, с которым заключается договор. На форме присутствует ссылка на отчет «Взаиморасчеты с контрагентами», который служит для анализа финансовых взаимоотношений с контрагентами, а также для получения информации о денежных средствах, отданных под отчет по конкретному контрагенту.

    В левом нижнем углу форм присутствуют ссылки на элемент справочника «Контрагенты и контакты».

Способ вызова

Вызывается из меню "Справочники --> Идентификационные --> Договоры взаиморасчетов".

Дополнительная информация

При открытии формы списка из полей ввода документов автоматически устанавливается отбор по дате: ДатаНачала <= ДатаДокумента.

Также устанавливается значение в поле отбора: ДатаКонца >= ДатаДокумента, но в отборе это поле не участвует.

При таких настройках активируется режим раскрашивания строк табличной части в разные цвета, в зависимости от того, подходит ли данный договор по истечению срока действия:

  • зеленый - дата окончания действия договора не задана;
  • синий - дата окончания действия договора больше либо равна дате документа;
  • красный - дата окончания действия договора меньше даты документа.
