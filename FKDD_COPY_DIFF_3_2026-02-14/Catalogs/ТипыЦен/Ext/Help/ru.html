Справочник "Типы цен"

Назначение

В справочник "Типы цен" необходимо ввести используемые компанией названия типов цен (например: "розничная", "оптовая", "дилерская" и т.д.). Для каждой позиции номенклатуры может быть назначено столько цен, сколько их хранится в справочнике "Типы цен". Для каждого договора, оформляемого с контрагентом можно назначить определенный тип цен продажи и покупки по умолчанию. Для типа цены может быть задана любая валюта. Задание валюты не зависит от способа формирования цены. Порядок округления также может быть задан произвольным образом для каждого типа цены.  Округление цен может производиться в большую или меньшую сторону.

В конфигурации предусмотрена возможность динамического расчета цен на основании заданной наценки и базового типа цен. Для того, чтобы цены динамически рассчитывались, необходимо в типе цен выбрать "Цены рассчитываются автоматически от цен базового типа", указать процент скидки (наценки) и тип того базового типа цен, на основании которого они будут рассчитываться. Если валюта базового типа цены и расчетного типа цены не совпадают, то значение цены пересчитывается на дату записи цены по курсу, определенному в справочнике валют.

Для просмотра истории цен номенклатуры в разрезе типов цен и поставщиков предусмотрен регистр сведений «Цены».

Вызов: "Перейти --> Цены".

Относится к подсистеме: «Номенклатурные».

Предопределенные элементы

  • Основной тип цен закупки. Тип цен закупки по умолчанию. Наименование можно изменить.
  • Основной тип цен продажи. Тип цен продажи по умолчанию. Наименование можно изменить.
  • Нормативная цена. Тип цен для произведенного товара. Наименование можно изменить.

Параметры

Длина кода: 8, тип кода: Строка.

Длина наименования: 25.

Не иерархический.

Реквизиты

  • Для цен товаров. Тип значения: «Булево». Флаг указывает, что тип цен используется для учета цен номенклатуры. 
  • Для цен работ. Тип значения: «Булево». Флаг указывает, что тип цен используется для учета цен по работам в заказ-нарядах.
  • Для цен автомобилей. Тип значения: «Булево». Флаг указывает, что тип цен используется для учета цен автомобилей.
  • Расчет цен. Тип значения: «Булево». Флаг представляет собой выбор между двумя расчетами цен: "Цены назначаются и хранятся для каждой позиции номенклатуры" и "Цены автоматически рассчитываются от базового типа цен". При выборе первого вида расчета цен становится активным флаг "Регистрировать цены при поступлении". При выборе второго вида расчета становится активным поле "Базовый тип цен".
  • Базовый тип цен. Реквизит является обязательным для заполнения. Ссылается на справочник  «Типы цен». Тип цен, от которого рассчитываются цены данного типа, если не выбран, то цены задаются вручную. Поле активно при выборе автоматицеского расчета цены по базовой цене ("Расчет цен").
  • Процент наценки. Тип значения «Число». Процент наценки или скидки на базовый тип цен.
  • Валюта цены. Реквизит является обязательным для заполнения. Ссылается на справочник «Валюты». Валюта цены по умолчанию.
  • В валюте учета. Тип значения: «Булево». Истина - вести учет в валюте учета номенклатуры, ложь - в валюте типа цены.
  • Вид оплаты. Реквизит является обязательным для заполнения. Ссылается на перечисление «Виды оплаты». Вид оплаты. Возможные значения: наличный расчет, безналичный расчет, произвольная оплата.
  • Цена включает НДС. Тип значения: «Булево». Истина - цена включает НДС, ложь - цена НДС не включает.
  • Метод округления. Тип значения: «Булево». Всегда в большую сторону - при округлении 0.04 будет округлена до 0.1, округление по арифметическим правилам - округление будет происходить по арифметическим правилам. 
  • Точность. Тип значения: «Число». Количество значащих знаков цены после запятой.
  • Закупка. Тип значения: «Булево». Признак использования в качестве цен закупки.
  • Продажа. Тип значения: «Булево». Признак использования в качестве цен продажи.
  • Регистрировать цены при поступлении. Тип значения: «Булево». Автоматически регистрировать цены при поступлении товара на склад. Флаг активен при назначении и хранении цены для каждой позиции номенклатуры ("Расчет цен"). 
  • Включать в прайс-лист. Тип значения: «Булево». Признак отображения данного типа цен при печати прайс-листа.
  • Аналитика учета цен. Реквизит является обязательным для заполнения. Ссылается на перечисление «Алгоритм получения цены». Аналитика, по  которой учитываются цены.
  • Комментарий. Тип значения: «Строка». Любая дополнительная информация.
  • Только продажа. Тип значения: «Булево». Истина - в форме списка отображаются типы цен, у которых установлен флаг "Документы закупки товаров".
  • Только закупка. Тип значения: «Булево». Истина - в форме списка отображаются типы цен, у которых установлен флаг "Документы продажи".

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов. На форме присутствуют флаги "Только закупка" и "Только продажа", при установлении в истинность которых происходит отображение типов цен у которых установлены флаги "Документы закупки товаров" и "Документы продажи" соответственно
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов и параметров. С помощью формы указываются порядок назначения цен и порядок округления цен.

Способ вызова

Вызывается из меню "Справочники --> Номенклатурные --> Типы цен".
