Справочник "Рассылки отчетов"

Назначение

Рассылки отчетов предназначены для автоматического формирования отчетов и доставки сформированных отчетов получателям. Подготовленные рассылки отчетов могут выполняться автоматически по расписанию или запускаться вручную.

Относится к подсистеме: «Отчеты».

Параметры

Длина кода: 0.

Длина наименования: 150.

Не иерархический.

Реквизиты

  • Наименование. Наименование рассылки или группы рассылок.
  • Подготовлена. флаг готовности рассылки к выполнению. Включение активизирует дополнительные проверки заполнения реквизитов.
  • Отправлять. Вид рассылки, определяющий цель:
    • Отчеты указанным получателям - Рассылка по получателям. Все способы доставки (по электронной почте и публикация).
    • Свой отчет для каждого получателя - Рассылка получателям, по персональному отбору. Способ доставки только по электронной почте.
    • Только мне (личная рассылка автора) - В качестве получателя автор. Способ доставки только по электронной почте.
  • Получатели. Вид получателей рассылки отчетов.
    • Контрагенты - Получателями отчетов будут контрагенты.
    • Пользователи - Получателями отчетов будут пользователи.
  • Автор. Ссылается на справочник «Пользователи». Получатель личной рассылки.
  • Почтовый адрес. Ссылается на справочник «Виды контактной информации». Почтовый адрес автора.

Страница "Отчеты":

  • Таблица "Отчеты". Содержит список «Вариантов отчета» для отправки.
  • Настройки. Настройки «Варианта отчета».
  • Вариант периода. Вариант настройки периода отчета, для текущей рассылки отчета.
  • Поле отбора. Имя поля отбора, по которому определяется связь данных с получателями. Данный реквизит относится к персональной рассылке.

Страница "Расписание":

  • Выполнять по расписанию. Признак включения автоматической рассылки отчетов.
  • Периодичность расписания. Вариант периодичности автоматической рассылки отчетов.
    • Ежедневно - Рассылка будет выполняться с заданной частотой в днях.
    • Еженедельно - По выбранным дням недели.
    • Ежемесячно - По выбранным месяцам в требуемый день месяца.
    • Расширенные параметры - Позволяет детально настроить каждый параметр расписания.
  • Время начала. Время запуска рассылки.
  • Пользователь. Имя пользователя информационной базы. При выполнении по расписанию отчеты формируются с правами этого пользователя.

Подробнее по настройке расписания можно прочесть в содержании справки в разделе «Расписание регламентного задания».

Страница "Доставка":

Содержимое страницы может меняться, в зависимости от вида рассылки. 

  • Публиковать. Признак необходимости публикации рассылки.
    • В сетевой каталог - отчеты будут сохранены в каталог файловой системы. Запись производится с сервера, поэтому рекомендуется указывать сетевой путь в формате UNC, например: \\server\folder\.
    • На FTP ресурс - отчеты будут сохранены на ftp сервере.
  • Сетевой каталог windows. Путь к сетевому каталогу в windows.
  • Сетевой каталог linux. Путь к сетевому каталогу в Linux.
  • Проверить. Проверка публикации путем записи и удаления тестового файла. Тест считается успешным если ошибок не возникло.
  • Отправлять по электронной почте. Признак использования электронной почты для отправки отчетов (или уведомлений).
  • Проверить электронную почту. Проверка учетной записи электронной почты путем отправки тестового сообщения. Тест считается успешным если ошибок не возникло.
  • От. Учетная запись отправителя электронной почты.
  • Тема и текст письма. могут содержать шаблоны параметров рассылки:
    • [ДатаВыполнения] - дата рассылки. Параметр может быть указан в «Имени архива».
    • [Автор] - пользователь, запустивший рассылку.
    • [Получатель] - представление получателя рассылки. Доступен только для вида рассылки «Свой отчет для каждого получателя».
    • [НаименованиеРассылки] - наименование рассылки. Параметр может быть указан в «Имени архива».
    • [СформированныеОтчеты] - список отчетов и форматов, в которые они были сохранены.
    • [ЗаголовокСистемы] - заголовок информационной базы 1С:Предприятие для всех пользователей.
    • [СпособДоставки] - представление ресурса, на котором был опубликован отчет.
  • Почтовый адрес. Ссылается на справочник «Виды контактной информации». Вид почтового адреса получателей.
  • Список получателей. Список получателей отчетов. Таблица не доступна для вида рассылки «Только мне (личная рассылка автора)».
    • Получатель - «Контрагенты» или «Пользователи», для которых будут отправлены отчеты. Можно выбирать элементы или группы получателей.
    • Исключен. Признак убранного получателя из рассылки.

Страница "Дополнительно":

  • Форматы сохранения отчетов. Определяет форматы, в которых будут сохранены отчеты по умолчанию. В списке отчетов, каждого отчета можно задать свой набор форматов.
  • Архивировать в ZIP. Признак архивации сформированных отчетов. Рекомендуется включать архивацию если отчеты отправляются по электронной почте.
  • Имя архива. Шаблон для создания имени архива отчета.
  • Пароль архива. Пароль к архиву.
  • Адрес ответа. E-mail адрес, на который будут высылаться ответы на рассылку.
  • Отправлять письма получателям через скрытые копии. Признак отправки писем получателям, через скрытые копии.
  • Группа. Группа, к которой принадлежит рассылка.
  • Список форматов рассылки. Список форматов сохранения для каждого отчета.
    • Отчет. «Вариант отчета» .
    • Форматы. Набор форматов сохранения отчета.
    • Отправлять, если пустой. Признак отправки сформированного отчета, даже если он пустой. Отчеты с графикой всегда считаются не пустыми.

Команды:

  • Выполнить сейчас. Позволят выполнить выбранные рассылки или группы рассылок от имени текущего пользователя. Выполняться будут только те рассылки, у которых установлен флажок подготовленности.

Работа в клиент-серверном режиме:

При работе информационной базы в клиент-серверном режиме необходимо иметь в виду, что рассылка выполняется на сервере 1С:Предприятия.

Путь к одному ресурсу может отличаться для клиента и сервера, в зависимости от их сетевого расположения и операционных систем.

Если сервер 1С:Предприятия 8.2 работает под Windows, то необходимо убедиться, что сетевой каталог Windows доступен для чтения и записи пользователю операционной системы, от имени которого запущен сервер 1С:Предприятия. Если это не так, то необходимо дать этому пользователю необходимые права, или выбрать другого пользователя для запуска сервера 1С:Предприятия с необходимыми правами к сетевому каталогу. Эти же рекомендации актуальны и для доступа к ресурсам FTP и электронной почты.

Поле сетевой каталог Linux предназначено для серверов 1С:Предприятия 8.2 работающих под управлением ОС Linux. В этом поле можно указать только локальный путь, например /home/path. Чтобы использовать UNC путь, нужно сделать на него ссылку. Для этого под правами администратора (root) сделайте следующее:

mkdir folder - вместо folder напишите имя каталога. например в каталоге /home делаем mkdir shared - создаст каталог shared с полным путем /home/shared.

mount -t cifs //server/share folder -o //domain/user, password=pwd
- где //server/share - UNC путь к другому компьютеру , folder - путь каталога на Linux компьютере, например \home\shared, //domain/user - имя домена/пользователя, если нужно, pwd - пароль пользователя (если
нужно).

Если имя и пароль не нужны, команда выглядит так:
mount -t cifs //server/share folder

После этого в пути к тому указываем folder, например /home/shared

Формы

  • Форма списка. Основная форма списка. Используется для отображения списка элементов.
  • Форма элемента. Основная форма элемента. Используется для заполнения реквизитов, просмотра и редактирования варианта отчета.
  • Параметры FTP. Основная форма выбора. Используется для отображения выбора элементов.
  • Форма выбора группы. Основная форма выбора. Используется для отображения выбора элементов.
  • Форма группы. Основная форма выбора. Используется для отображения выбора элементов.
  • Форма подбора вариантов. Основная форма выбора. Используется для отображения выбора элементов.
  • Форма Выбора. Основная форма выбора. Используется для отображения выбора элементов.

Способ вызова

Вызывается из меню "Справочники --> Структура компании --> Рассылки отчетов".
