Справочник "Учетные записи электронной почты"

Назначение

Справочник предназначен для использования во встроенном почтовом клиенте и содержит настройки учетных записей электронной почты.

Пользователь регистрирует необходимые учетные записи в справочнике. При этом назначается ответственный за учетную запись, пользователь, который может изменять ее параметры, назначать уровень доступа другим пользователям.

Относится к подсистеме: «Сервис».

Параметры

Длина кода: 8, тип кода: Строка.

Длина наименования: 100.

Не иерархический.

Реквизиты

  • E-mail. Реквизит является обязательным для заполнения. Тип значения: «Строка». Адрес электронной почты.
  • Представление. Реквизит является обязательным для заполнения. Тип значения: «Строка». Представление пользователя почтового ящика, используется для заполнения по умолчанию поля "От" в новых электронных письмах. Автоматически копируется содержание поля "E-mail".
  • Имя (от). Тип значения: «Строка». Представление пользователя, добавляется при отправке к адресу электронной почты в поле "От".
  • SMTP сервер. Тип значения: «Строка». Реквизит является обязательным для заполнения. Тип значения: «Строка». Интернет адрес SMTP сервера.
  • Порт SMTP. Реквизит является обязательным для заполнения. Тип значения: «Число». Тип значения: «Число». Порт SMTP сервера. По умолчанию 25. 
  • POP3 сервер. Реквизит является обязательным для заполнения. Тип значения: «Строка». Тип значения: «Строка». Интернет адрес POP3 сервера.
  • Порт POP3. Реквизит является обязательным для заполнения. Тип значения: «Число». Тип значения: «Число». Порт POP3 сервера. По умолчанию 110.
  • Переключатель POP3 / IMAP. Выбор между протоколами получения почты POP3 и IMAP. Для IMAP используются те же логин и пароль, что и для POP3. При изменении проверьте правильность настройки сервера получения почты и порта. (Например: IMAP.<имя сервера>.ru, POP.<имя сервера>.ru).
  • Аутентификация SMTP. Ссылается на перечисление: «Способ аутентификации SMTP». Проверка подлинности пользователя для SMTP сервера. Может принимать следующие значения: без аутентификации, Login, Plain, CramMD5, по умолчанию. При виде аутентификации "без аутентификации" поля "Пользователь", "Пароль" для SMTP сервера и флаг "Аутентификация POP прежде SMTP" не активны.
  • Аутентификация POP3. Ссылается на перечисление: «Способ аутентификации POP3». Виды проверки подлинности пользователя для POP3 сервера. Может принимать следующие значения: обычная, APOP, CramMD5
  • Пользователь (логин).  Тип значения: «Строка». Логин пользователя почтового ящика при SMTP и POP3 аутентификациях. Реквизит является обязательным для заполнения при POP3 аутентификации. При виде аутентификации SMTP "без аутентификации" поле для SMTP сервера не доступно.
  • Пароль. Тип значения: «Строка». Тип значения: «Строка». Пароль пользователя почтового ящика. Реквизит является обязательным для заполнения при POP3 аутентификации. При виде аутентификации SMTP "без аутентификации" поле для SMTP сервера не доступно.
  • Аутентификация POP прежде SMTP. Тип значения: «Булево». Признак того, что POP аутентификация должна произойти до аутентификации SMTP на почтовом сервере. При виде аутентификации SMTP "без аутентификации" поле не доступно.
  • Время ожидания сервера. Реквизит является обязательным для заполнения. Тип значения: «Число». Время ожидания ответа сервера при попытки соединения. По умолчанию 60 секунд. Минимальное допустимое значение 1 секунда.
  • Удалять письма с сервера. Тип значения: «Булево». Признак удаления почтового сообщения с почтового сервера после его получения. При активности флага не активны флаг "Срок хранения писем на сервере" и поле "Количество дней удаления писем с сервера". 
  • Оставлять письма на сервере. Тип значения: «Булево». Признак хранения почтового сообщения на почтовом сервере после его получения.
  • Срок хранения писем на сервере. Тип значения: «Булево». Если флаг установлен в истину то письма будут храниться на сервере указанное количество дней, иначе письма будут храниться на сервере пока пользователь сам не удалит их. Поле активно при истинности флага "Оставлять письма на сервере". 
  • Количество дней до удаления писем с сервера. Тип значения: «Число». Количество дней до удаления писем с сервера. По умолчанию один день.  Минимальное допустимое значение 1 день. Поле активно при истинности флага "Оставлять письма на сервере".
  • Авто получение/отправка писем. Тип значения: «Булево». Флаг авто получения / отправки сообщений. При истинности флага становятся активными флаг "Оповещать об автополучении почты" и поля "Интервал автополучения/отправки писем", "Действие", "Ответственный".
  • Оповещать об автополучении почты. Тип значения: «Булево». Включать учетную запись в напоминание об автополучении почты. Флаг активен при истинности флага "Авто получение/отправка почты". 
  • Интервал автополучения/отправки писем. Тип значения: «Число». Интервал автополучения / отправки сообщений. По умолчанию 15 минут. Минимальное допустимое значение 1 минута. Поле активно при истинности флага "Авто получение/отправка почты".
  • Действие. Ссылается на перечисление «Виды действий автополучения/отправки электронных писем». Действие при автополучении или отправки сообщений. Может принимать следующие значения: получение, отправка, получение и отправка. Поле активно при истинности флага "Авто получение/отправка почты".
  • Ответственный. Ссылается на справочник «Пользователи». Ответственный за авто получение/отправку писем. Поле активно при истинности флага "Авто получение/отправка почты".
  • Автосохранение писем. Тип значения: «Булево». Признак автосохранения писем. При истинности флага становится активным поле "Интервал автосохранения писем".
  • Интервал автосохранения писем. Тип значения: «Число». Интервал автосохранения писем. По умолчанию 30 секунд. Минимальное допустимое значение 1 секунда. Поле активно при истинности флага "Автосохранение писем".
  • Автоматическая установка пометки "рассмотрено". Тип значения: «Булево». Признак автоматической установки пометки "рассмотрено". При истинности флага становится активным поле "Интервал автоматической установки отметки рассмотрено".
  • Интервал автоматической установки отметки рассмотрено. Тип значения: «Число». Интервал автоматической установки отметки "Рассмотрено". По умолчанию 3 секунды. Минимальное допустимое значение 1 секунда. Поле активно при истинности флага "Автоматическая установка пометки "рассмотрено"".
  • Автоматическая установка пометки "рассмотрено" при ответе. Тип значения: «Булево». Автоматическая установка пометки "рассмотрено" при ответе.
  • Автоматическое получение писем при старте системы за последние. Тип значения: «Булево». Флаг отправки/получения сообщений при старте системы. При истинности флага при старте системы для данной учетной записи будут получены письма за указанное количество дней. Если количество дней 0, будут получены все письма без ограничений, если 1 - за последний день, и т.д. Данный отбор работает только для получения по протоколу IMAP. Не рекомендуется устанавливать для протокола POP3 (если это новая учетная запись с большим количеством писем на сервере и не включено удаление писем на сервере система при старте может долго загружать письма).
  • Учетная запись для общего доступа. Тип значения: «Булево». Признак, что это учетная запись для общего доступа.

Табличная часть «Пользователи»

  • Пользователь. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Пользователи».
  • Чтение. Тип значения: «Булево». Флаг разрешения чтения учетной записи. Всегда истина.
  • Администрирование. Тип значения: «Булево». Флаг разрешения администрирования учетной записи.
  • Редактирование. Тип значения: «Булево». Флаг разрешения создания/редактирования писем данной учетной записи.
  • Транспорт. Тип значения: «Булево». Флаг разрешения получения/отправки писем данной учетной записи.
  • Учетная запись по умолчанию. Тип значения: «Булево». Признак что это учетная запись по умолчанию данного пользователя.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов. На форме присутствуют закладки:
    • Закладка "Параметры подключения". Закладка содержит параметры подключения к сервером SMTP и POP. На ней также задаются логины и пароли пользователей и время ожидания ответа сервера.
    • Закладка "Управление почтой". Закладка содержит реквизиты управления почтой такие как удаление писем с сервера, автополучение/доставка писем, автосохранение писем, пометка писем как "прочитано".
    • Закладка "Группы писем". Закладка доступна только после записи элемента. При подключении к почтовому ящику на эту закладку копируются группы писем, которые установлены на сервере для этого ящика. 
    • Закладка "Подпись". На закладке представлено поле подписи письма. Тип значения «Строка». Сохраненная подпись будет автоматически вставляться в каждое отправленное письмо.
    • Закладка "Доступ". Закладка содержит табличную часть "Пользователи", в которой содержится список пользователей с их правами доступа к учетной записи.

На форме присутствует кнопка "Проверить параметры подключения", при нажатии на которую проверяются настройки учетной записи. При возникновении ошибок, они выводятся в окне сообщений.

Способ вызова

Вызывается из меню "Справочники --> Структура компании --> Учетные записи электронной почты"
