Документ "Техническое диагностирование"

Назначение

Документ "Техническое диагностирование" предназначен для фиксации факта проведения диагностирования транспортных средств. Техническое диагностирование можут быть как первичным, так и повторным. Документ вводится только на основании других документов. Для первичного диагностирования ввод осуществляется на основании документа «Договор проведения технического осмотра», для повторного диагностирования - на основании первичного «Техническое диагностирование».

При проведении документ формирует хозяйственные операции:

  • Техническое диагностирование. Первичное диагностирование. Данная операция устанавливается автоматически при вводе на основании документа «Договор проведения технического осмотра».
  • Повторное техническое диагностирование. Повторное диагностирование. Данная операция устанавливается автоматически при вводе на основании документа «Техническое диагностирование» с проваленным ТО.
  • Дубликат технического диагностирования. Дубликат диагностирования. Данная операция устанавливается автоматически при вводе на основании документа «Техническое диагностирование» с пройденным ТО. 

Относится к подсистеме: «Технический осмотр».

Параметры

Длина номера: 15 (см. "Дополнительная информация").

Периодичность номера: Год.

Тип номера: Строка.

Вводится на основании: «Договор проведения технического осмотра», «Техническое диагностирование».

Объекты, вводимые на основании: «Техническое диагностирование».

Реквизиты

  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагенты и контакты». Контрагент, предоставляющий автомобиль для проведения диагностирования.
  • Договор. Реквизит является обязательным для заполнения. Ссылается на справочник «Договоры взаиморасчетов». Договор, по которому ведутся взаиморасчеты с контрагентом.
  • Документ основание. Ссылается на документы «Договор проведения технического осмотра», «Техническое диагностирование». Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов (по кнопке ).
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.
  • Автомобиль. Реквизит является обязательным для заполнения. Ссылается на справочник «Автомобили». Автомобиль подлежащий диагностированию.
  • Тормоза. Реквизит является обязательным для заполнения. Ссылается на перечисление «Тип привода тормозной системы». Тип привода тормозной системы автомобиля. 
  • Шины. Реквизит является обязательным для заполнения. Тип "Строка". Марка шин автомобиля.
  • Тип топлива. Реквизит является обязательным для заполнения. Ссылается на перечисление «Типы топлива». Тип топлива используемый автомобилем.
  • Масса. Реквизит является обязательным для заполнения. Тип "Число". Масса автомобиля без нагрузки.
  • Макс. масса. Реквизит является обязательным для заполнения. Тип "Число". масимально. Максимально разрешенная масса автомобиля.
  • Срок повторного ТО. Реквизит является обязательным для заполнения. Тип значения: «Дата». Не доступен для хоз.операции "Повторное техническое диагностирование". Доступен при условии, что не все требования выполнены. На повторный технический осмотр даётся 20 дней от даты проведения первичного технического осмотра.
  • Срок окончания срока действия ТО. Реквизит является обязательным для заполнения. Тип значения: «Дата». Доступен при условии, что все требования выполнены. Срок окончания действия талона технического осмотра согласно Федеральному закону "О техническом осмотре транспортных средств" от 1 июля 2011 года №170-ФЗ г.Москва, статья 15. Формируется прибавлением к дате документа числа месяцев, равного периодичности прохождения технического осмотра. Периодичность зависит от года выпуска автомобиля, указанного в справочнике «Автомобили», и категории автотранспортного средства, указанного в справочнике «Модели» для данного автомобиля, и вычисляется по данной таблице:

Табличная часть «Требования»

  • Требование к транспортному средству. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Требования к транспортным средствам». Требование к транспортному средству.
  • Выполнено. Тип значения: «Булево». Признаку соответствия транспортного средства данному требованию.
  • Значение параметра. Тип значения: «Число». Фактическое значение требования, выявленное в ходе проведения диагностирования (если такое необходимо).
  • Значение минимальное. Тип значения: «Число». Минимальное значение требования для данной категории траспортных средств (если такое необходимо).
  • Значение максимальное. Тип значения: «Число». Максимальное значение требования для данной категории траспортных средств (если такое необходимо).

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора документов данного типа.
  • Форма документа. Основная форма документа. Используется для заполнения обязательных реквизитов.

Печатные формы

  • Диагностическая карта. Диагностическая карта проведения первичного технического осмотра.
  • Диагностическая карта (незаполненная). Незаполненная диагностическая карта проведения технического осмотра для ручного заполнения.
  • Диагностическая карта (повторный осмотр). Диагностическая карта проведения повторного технического осмотра.

Способ вызова

Вызывается из меню "Документы --> Автосервис --> Техническое диагностирование".

Движения

  • Регистр сведений «Техническое диагностирование».

Дополнительная информация

Регистрационный номер, присваиваемый диагностической карте, состоит из пятнадцати значащих разрядов: Х1Х2Х3Х4Х5Х6Х7Х8Х9Х10Х11Х12Х13Х14Х15

1) первый, второй, третий, четвертый, пятый разряды (X1Х2Х3Х4X5) указывают на номер Оператора либо Организации в Реестре операторов технического осмотра (например: 00101);

2) шестой, седьмой, восьмой разряды (Х6Х7Х8) указывают на порядковый номер конкретного пункта технического осмотра (далее – ПТО), принадлежащего Оператору либо Организации (например: 001). Данный порядковый номер ПТО присваивается Оператором либо Организацией самостоятельно;

3) девятый, десятый разряды (Х9Х10) указывают на год проведения технического осмотра (например: «12»  – 2012 год);

4) одиннадцатый, двенадцатый, тринадцатый, четырнадцатый, пятнадцатый разряды (Х11Х12Х13Х14Х15) указывают порядковый номер диагностической карты в нарастающем порядке, от номера 00001 до номера 99999, по каждому конкретному ПТО за один год.

Первого января каждого года порядковый номер диагностической карты, указываемый в разрядах Х11Х12Х13Х14Х15, обнуляется.

 

Раскраска строк в форме списка и расшифровка цвета

    • Зеленый цвет - документ готов к отправке в ЕАИС ТО.
    • Серый цвет     - документ отправлен в ЕАИС ТО.
    • Черный цвет   - нет сведений об этом документе.
