Документ «Жалоба клиента»

Назначение

Документ предназначен для оформления жалобы клиента. Позволяет фиксировать и обрабатывать жалобы клиентов.

Параметр

Длина номера: 10.

Периодичность номера: Непериодический.

Тип номера: Строка.

Вводится на основании: «Рассылка», «Заказ-наряд», «Заявка на ремонт», «Реализация товаров», «Заказ покупателя», «Заказ на автомобиль», «Реализация автомобилей», «Событие», «Рабочий лист», «Опрос».

Объекты, вводимые на основании: «Событие», «Электронное письмо».

Реквизиты

  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник "Пользователи". Автор документа - пользователь, породивший документ.
  • Дата и время реакции. Тип значения: Дата. Дата и время, до наступления которых необходимо среагировать на жалобу - связаться с клиентом первый раз. Рассчитывается как "Дата и время регистрации жалобы" + "Значение настройки времени реакции на жалобу".
  • Дата и время разбора. Тип значения: Дата. Дата и время, до наступления которых необходимо разобрать жалобу. Рассчитывается как "Дата и время регистрации жалобы" + "Значение настройки времени разбора жалобы".  
  • Действия по исправлению. Тип значения: Строка. Действия, предпринятые сотрудниками для исправления возникшей ситуации.
  • Действия по предотвращению. Тип значения: Строка. Действия, предпринятые сотрудниками для исключения возникновения подобных ситуаций в будущем. 
  • Договор взаиморасчетов. Ссылается на справочник "Договоры взаиморасчетов". Договор взаиморасчетов с клиентом.
  • Документ основание. Документ, на основании которого был введен рабочий лист отдела страхования.
  • Жалоба справедлива. Тип значения: Булево. Признак обоснованности жалобы.
  • Контактная информация контактного лица контрагента. Тип значения: Строка. Контактная информация контактного лица контрагента.
  • Контактная информация контрагента. Тип значения: Строка. Контактная информация контрагента.
  • Контактное лицо контрагента. Ссылается на справочник "Контрагенты". Контактное лицо контрагента.
  • Комментарий. Тип значения: Строка. Произвольный комментарий к документу.
  • Клиент удовлетворен. Тип значения: Булево. Признак согласия клиента.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник "Контрагенты". Клиент.
  • Мнение клиента. Тип значения: Строка. Мнение клиента.
  • Мнение ответственного. Тип значения: Строка. Мнение ответственного лица.
  • Мнение сотрудника. Тип значения: Строка. Мнение сотрудника.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник "Организации". Идентификатор юридического лица, от которого выписан документ.
  • Ответственный. Реквизит является обязательным для заполнения. Ссылается на справочник "Сотрудники". Ответственный за обработку жалобы клиента.
  • ПодразделениеКомпании. Реквизит является обязательным для заполнения. Тип значения: Подразделения компании .. ПодразделениеКомпании
  • Проект. Ссылается на справочник "Проекты". Проект документа
  • Состояние. Реквизит является обязательным для заполнения. Ссылается на перечисление "Состояния жалоб клиентов". Состояние жалобы клиента.
  • Сотрудник. Ссылается на справочник "Сотрудники". Сотрудник.
  • Суть. Реквизит является обязательным для заполнения. Тип значения: Строка. Суть жалобы.
  • Операция. Ссылается на справочник "Хоз. операции". Выполняемая документом операция.

Табличная часть «Категории»

  • Категория. Ссылается на справочник "Категории жалоб клиентов". Категория жалобы клиента.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора документов данного типа.
  • Форма документа. Основная форма документа. Используется для заполнения обязательных реквизитов.

Способ вызова

Вызывается из меню "Документы --> Взаимоотношения с клиентами --> Жалоба клиента".
