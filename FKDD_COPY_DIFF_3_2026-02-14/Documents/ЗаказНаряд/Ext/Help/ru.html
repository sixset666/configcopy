Документ "Заказ-наряд"

Назначение

Документ "Заказ-наряд" предназначен для фиксации факта выполнения ремонта автомобиля и реализации деталей для выполнения данного ремонта. В процессе подготовки документа документ находится в непроведенном состоянии. Перед закрытием заказ-наряда его требуется перевести в состояние "Выполнен". На этом этапе осуществляется проверка возможности закрытия заказ-наряда (без учета оплаты).

Список хозяйственных операций:

  • Заказ-наряд. Хозяйственная операция списывает кредиторскую задолженность плательщика по сделке заказ-наряда и начисляет дебиторскую задолженность плательщика перед компанией на регистре «Взаиморасчетов компании», приходует сумму и себестоимость использованных запасных частей на регистре «Продажи», приходуется сумма розничной наценки на регистре «Доходов и расходов» компании. Помимо этого производится списание запасных частей с регистра «Товары компании в производстве», требуемых для выполнения работ по заказ-наряду и запись в регистр «Выработка сотрудников» информации о выработке исполнителей по данному заказ-наряду.

Относится к подсистеме: «Автосервис».

Параметры

Длина номера: 10

Периодичность номера: Непериодичный.

Тип номера: Строка

Вводится на основании: «Заявка на ремонт».

Объекты, вводимые на основании: «Заказ покупателя», «Приходный кассовый ордер», «Перемещение товаров в производство», «Извлечение товаров из производства», «Счет на оплату», «Счет фактура выданный», «Ввод остатков товаров в производстве».

Реквизиты

  • Ссылка на заказ-наряд. Ссылается на документ «Заказ-наряд». Ссылка необходима для включения документа в производственную последовательность.
  • Документ основание. Ссылается на документ «ЗаявкаНаРемонт». Документ, на основании которого введен данный заказ-наряд, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов (по кнопке ).
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ.
  • Менеджер. Ссылается на справочник «Сотрудники». Сотрудник, заключивший сделку с контрагентом. 
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Регламентированный учет. Тип значения: «Булево». Флаг отражения документа в регламентированном учете.
  • Комментарий. Произвольный комментарий к документу.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагенты и контакты». Плательщик по заказ-наряду.
  • Договор. Реквизит является обязательным для заполнения. Ссылается на справочник «Договоры взаиморасчетов». Договор по которому ведутся взаиморасчеты с контрагентом.
  • Карточка. Ссылается на справочник «Карточки». Дисконтная карта.
  • Валюта. Реквизит является обязательным для заполнения. Ссылается на справочник «Валюты». Валюта документа.
  • Курс. Реквизит является обязательным для заполнения. Курс валюты документа.
  • Тип Цен. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы цен». Тип Цен, в которых отгружаются детали для выполнения работ по заказ-наряду.
  • Маркетинговая программа. Ссылается на справочник «Маркетинговые программы». Маркетинговая программа, в рамках которой осуществляется ремонт.
  • Скидка/наценка. Ссылается на справочник «Типы скидок и наценок». Скидка/наценка на запчасти по заказ-наряду.
  • Скидка/наценка на работы. Ссылается на справочник «Типы скидок и наценок». Скидка/наценка на работы по заказ-наряду.
  • Вид оплаты. Вид оплаты заказ-наряда.
  • Сумма. Сумма документа.
  • Сумма номенклатуры документа. Сумма запчастей по заказ-наряду.
  • Сумма работ документа. Сумма работ по заказ-наряду.
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.
  • Автомобиль. Реквизит является обязательным для заполнения. Ссылается на справочник «Автомобили». Ремонтируемый автомобиль.
  • Пробег. Тип значения: «Число». Пробег автомобиля на дату начала работ по заказ-наряду.
  • Вид ремонта. Реквизит является обязательным для заполнения. Ссылается на справочник «Виды ремонта». Вид ремонта, производимого по заказ-наряду. При выборе вида ремонта задается вопрос пользователю о добавлении работ, назначененных по умолчанию для данного вида ремонта, в табличную часть "Работы".
  • Рекомендации. Рекомендации владельцу автомобиля.
  • Гарантии. Гарантии на выполненные работы и примененные детали.
  • Дата создания. Реквизит является обязательным для заполнения. Дата создания заказ-наряда.
  • Дата начала. Дата начала работ по заказ-наряду. Если заказ-наряд находится в состоянии "закрыт" - дата начала работ должна быть задана.
  • Дата окончания. Дата окончания работ по заказ-наряду. Если заказ-наряд находится в состоянии "закрыт" - дата окончания работ должна быть задана.
  • Состояние заказ-наряда. Реквизит является обязательным для заполнения. Текущее состояние заказ-наряда (заявка/начало работ/в работе/выполнен/закрыт).
  • Источник информации. Ссылается на справочник «Источники информации». Источник информации, по которому произошло обращение на выполнение работ.
  • Сервисная кампания. Ссылается на справочник «Сервисные кампании». Сервисная кампания, в рамках которой осуществляется ремонт.
  • Причина обращения. Причина обращения заказчика в ремонт.
  • Мастер. Ссылается на справочник «Сотрудники». Мастер, ответственный за выполнение ремонта.
  • Диспетчер. Ссылается на справочник «Сотрудники». Диспетчер, принявший заказ на выполнение ремонта.
  • Расчет цен деталей. Способ расчета цен деталей.
  • Заказчик. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагенты и контакты». Заказчик выполнения работ по заказ-наряду.
  • Исполнители для всех работ. Признак одинаковых исполнителей для всех работ по заказ-наряду.
  • Цех. Реквизит является обязательным для заполнения. Ссылается на справочник «Цеха». Основной цех проведения работ по заказ-наряду.
  • Автоматическое закрытие сделок. Если флажок установлен, то при проведении документа сначала будут оплачены все незакрытые сделки.
  • Пробег следующего ТО. Тип значения: «Число». Предельное значение пробега автомобиля при прохождении следующего техобслуживания.
  • Дата следующего ТО. Тип значения: «Дата». Предельная дата прохождения следующего техобслуживания.
  • Телефон. Телефон клиента, по которому он обратился в автосервис. Может быть выбран из списка, отредактирован или введен новый. Редактируется в диалоге. Для нового номера по умолчанию подставляется вид контактной информации "мобильный телефон". Если телефон с данным видом контактной информации уже задан в регистре для данного клиента, то задается вопрос о замене телефона либо выборе другого вида контактной информации для записи.

Табличная часть «Товары»

  • Наличие замен. В колонке отображается иконка "Замены" в случае, когда данная номенклатура имеет замены или является заменой другой номенклатуры. По нажатию на колонку открывается обработка «Подбор замен».
  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Деталь для выполнения работ по заказ-наряду.
  • Количество. Реквизит является обязательным для заполнения. Количество деталей.
  • Единица измерения. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Единицы измерения». Единица измерения деталей.
  • Коэффициент. Реквизит является обязательным для заполнения. Коэффициент пересчета по отношению к базовой единице номенклатуры.
  • Цена. Цена детали в валюте документа.
  • Сумма. Сумма всегда без скидки. НДС входит в сумму если это указано в типе цен. Сумма всегда равна Цена*Количество.
  • Ставка НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по номенклатурной позиции.
  • Сумма НДС. Сумма НДС по номенклатурной позиции.
  • Процент скидки. Процент скидки на деталь.
  • Сумма скидки. Сумма скидки на деталь.
  • Характеристика номенклатуры. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры.
  • Всего. Итоговая сумма по строке документа со всеми налогами.
  • Склад компании. Ссылается на справочник «Склады (места хранения) компании». Склад компании, откуда деталь поступает в производство.

Табличная часть «Работы»

  • Работа. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Автоработы». Работа для выполнения заказ-наряда.
  • Количество. Реквизит является обязательным для заполнения. Количество работ.
  • Нормочас. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Нормочасы». Нормочас работы.
  • Коэффициент. Реквизит является обязательным для заполнения. Норма времени выполнения одной работы (в часах).
  • Номер пакета. Номер пакета работ(редактирование происходит с помошью механизма распределения по пакетам).   
  • Цена. Цена работы в валюте документа.
  • Сумма. Сумма всегда без скидки. НДС входит в сумму если это указано в типе цен. Сумма всегда равна Цена*Количество*Коэффициент.
  • Ставка НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по работе.
  • Сумма НДС. Сумма НДС по работе.
  • Процент скидки. Процент скидки на работу.
  • Сумма скидки. Сумма скидки на работу.
  • Всего. Итоговая сумма по строке документа со всеми налогами.

Табличная часть «Исполнители»

  • Работа. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Автоработы». Работа, для которой назначен исполнитель.
  • Исполнитель. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Сотрудники». Исполнитель работы по заказ-наряду.
  • Цех. Реквизит является обязательным для заполнения. Ссылается на справочник «Цеха». Цех исполнения работы.
  • Процент. Реквизит является обязательным для заполнения. Процент участия исполнителя при выполнении данной работы.

Формы

  • Форма списка. Основная форма списка. Используется для отображения списка документов данного типа.
  • Форма документа. Основная форма документа. Используется для ввода и редактирования документов данного типа.
  • Форма строка товаров. Используется для выбора источника поступления деталей в производство.
  • Форма ввода дат документа. Форма используется для ввода дат начала и окончания работ по заявке на ремонт. Содержит раздельный ввод даты и времени соответствующих реквизитов.

Печатные формы

  • Бланк заявки. Бланк заявки на проведение ремонтных работ с указанием заказчика и ремонтируемого автомобиля, а так же причины обращения заказчика.
  • Рабочая заявка. Рабочая заявка на ремонт с перечнем работ и требующихся для их выполнения деталей.
  • Приемный акт. Приемный акт на проведение ремонтных работ.
  • Заказ-наряд. Перечень выполненных работ и израсходованных деталей компании и заказчика.
  • Заявка предварительная. Предварительная заявка на проведение ремонтных работ с указанием причины обращения.
  • Акт осмотра. Акт осмотра автомобиля при поступлении в ремонт с указанием комплектности и внешнего состояния. Для вывода в печатной формы схемы транспортного средства, отличной от по умолчанию, в модели автомобиля (или в группе моделей) необходимо добавить картинку с данной схемой. Идентификатор картинки при этом должен быть "СхемаТС".
  • Заказ-наряд договор. Договор с заказчиком на проведение ремонтных работ.
  • Акт приема. Акт приема автомобиля заказчиком после проведения ремонтных работ.
  • Сопроводительный лист. Сопроводительный лист на ремонт с перечнем работ и требующихся для их выполнения деталей.
  • Заявка на детали. Заявка на детали, требующиеся для проведения ремонтных работ.
  • Квитанция к заказ-наряду. Квитанция к заказ-наряду с перечнем выполненных работ, израсходованных деталей и их стоимостью.
  • Акт сдачи. Акт сдачи автомобиля заказчику после проведения ремонтных работ.

Способ вызова

Вызывается из меню "Документы --> Автосервис --> Заказ-наряд"

Движения

  • «Взаиморасчеты компании». Взаиморасчеты компании с контрагентами.
  • «Доходы и расходы». Доходы и расходы компании.
  • «Продажи». Учет продаж товаров и оказания услуг.
  • «Реализованные товары». Предназначен для накопления информации о товарах, принятых на реализацию, которые были нами проданы (или иным образом списаны), но еще не включены в состав отчета комитенту.
  • «Товары компании в производстве». Товары компании в производстве, используемые для выполнения работ.
  • «Выработка сотрудников». Выработка сотрудников по работам.

Дополнительная информация

Кнопка «Подбор Рекомендаций» вызывает обработку «Подбор рекомендаций» и служит для подбора рекомендуемой автоработы или номенклатуры для данного автомобиля.
Отображение иконки  возле реквизита «Автомобиль» показывает наличие рекомендаций для выбранного автомобиля.

По завершению работ (состояние заказ-наряда переходит в состояние "закрыт") выполняется его проведение и списание деталей из производства. Закрытие заказ-наряда возможно только при освоении всех деталей, требуемых для выполнения работ (детали должны быть перемещены в производство в полном объеме).

После того как заказ-наряд закрыт его редактирование блокируется. Право редактирования закрытых заказ-нарядов регулируется правом "Редактирование закрытых заказ-нарядов". По умолчанию данное право установлено в состояние "Ложь".

В случае ремонта без предварительной записи или для расширения ремонта (когда в заказ-наряд, созданный на основании заявки на ремонт, добавляются новые работы) работы заказ-наряда могут быть запланированы в АРМ Запись на ремонт. Для этого в списке "Перейти" добавлена кнопка перехода к АРМ (Alt+L). По нажатию на кнопку открывается АРМ с заполненным контекстом заказ-наряда. Выполняется контроль соответствия работ в табличной части "Работы" заказ-наряда и табличной части "Планирования" заявки на ремонт. Новые строки выделяются жирным шрифтом, отсутствующие в заказ-наряде строки помечаются как "зачеркнутые". Если ремонт выполнялся без предварительной записи, создается заполненная заявка на ремонт, она подставляется в поле "Документ-основание" заказ-наряда.

При изменении состояния заказ наряда, пользователю будет выведено окно отправки смс («Документ SMS»), который будет заполнен на основании текущего заказ-наряда. Такое поведение возможно лишь при включенном поле "Отправка уведомлений" в справочнике «ВидыРемонта». Шаблон смс будет выбран в соответствии с табличной частью «Настройка уведомлений» справочника «ВидыРемонта». Если шаблон не найден, то текст сообщения формироваться не будет, но остальные параметры отправки заполнятся.
