Документ "Событие"

Назначение

Документ предназначен для планирования и отработки контактов, презентаций, семинаров и прочих событий, связанных с управлением проектами и предконтрактными работами. Регистрация контактов с контрагентом позволяет увидеть историю взаимодействия с клиентом.  События фиксируют все формы взаимодействия с клиентом. Состоявшиеся события могут определять ваши последующие действия: на основании этого документа могут быть сформированы другие события.

Относится к подсистеме: «Сервис».

Параметры

Длина номера: 10.

Периодичность номера: Год.

Тип номера: Строка.

Вводится на основании: всех документов.

Объекты, вводимые на основании: «Событие», «Электронное письмо», «Корректировка».

Реквизиты

  • Тема. Реквизит является обязательным для заполнения. Тип значения: «Строка». Тема события.
  • Контрагент. Ссылается на справочник «Контрагенты и контакты». Контрагент.
  • Контактное лицо. Ссылается на справочник «Контрагенты и контакты». Контактное лицо.
  • Контактная информация. Тип значения: «Строка». Контактная информация.
  • Телефон. Телефон контрагента, по которому он обратился в автосервис. Может быть выбран из списка, отредактирован или введен новый. Редактируется в диалоге. Для нового номера по умолчанию подставляется вид контактной информации "мобильный телефон". Если телефон с данным видом контактной информации уже задан в регистре для данного контрагента, то задается вопрос о замене телефона либо выборе другого вида контактной информации для записи.
  • Вид события. Ссылается на перечисление «Виды событий». Вид события.
  • Состояние. Ссылается на перечисление «Состояния событий». Состояние.
  • Дата начала. Тип значения: «Дата». Дата начала.
  • Важность. Ссылается на перечисление «Важность». Приоритет.
  • Тип сообщения. Ссылается на перечисление «Типы сообщений». Тип сообщения.
  • Дата окончания. Тип значения: «Дата». Дата окончания.
  • Содержание. Тип значения: «Строка». Содержание.
  • Результат. Тип значения: «Строка». Результат события.
  • Документ основание. Ссылается на документы «Событие», «Электронное письмо». Документ, на основании которого создан новый документ.
  • Источник информации. Ссылается на справочник «Источники информации». Источник информации.
  • Продолжительность. Тип значения: «Число». Продолжительность события в часах.
  • Оборудование. Ссылается на справочник «Оборудование». SMS оборудование.
  • Номер отправителя. Тип значения: «Число». Номер отправителя.
  • Комментарий. Тип значения: «Строка». Комментарий документа.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение. 
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Наименование фирмы, в которой создан документ.
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Пользователь, создавший документ.
  • Формат текста. Ссылается на перечисление «Формат текста». Формат текста.
  • Проект. Ссылается на справочник «Проекты». Проект, к которому относится документ.

Табличная часть «Пользователи»

  • Пользователь. Ссылается на справочник «Пользователи». Пользователь.

Табличная часть «Сторонние лица»

  • Контрагент. Ссылается на справочник «Контрагенты и контакты». Контрагент.
  • Контактное лицо. Ссылается на справочник «Контрагенты и контакты». Контактное лицо.
  • Контактная информация. Тип значения: «Строка». Контактная информация.
  • Тип контактной информации. Ссылается на перечисление «Типы контактной информации». Тип контактной информации, например - адрес, телефон и другие.
  • Статус исходящего сообщения. Ссылается на перечисление «Статус SMS сообщений». Статус исходящего сообщения.
  • Дата получения. Тип значения: «Дата». Дата получения сообщения.
  • Дата события. Тип значения: «Дата». Дата совершения события.
  • Описание ошибки. Тип значения: «Строка». Описание ошибки.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора документов данного типа.
  • Форма документа. Основная форма объекта. Используется для заполнения обязательных реквизитов.
  • Форма записи события. Используется для обработки текущего события и планирования следующего события. Правила поледовательности выбора видов событий описано в справочнике «Описание последовательностей событий». Если установлен режим режим "Редактировать запланированное", то выбранный документ будет переведен в статус "Отменено" с выбранной причиной отмены и создано новое следующее событие.

Печатные формы

  • Событие. Простая печатная форма, выводит список участников события.

Способ вызова

Вызывается из меню  "Документы --> Событие".
