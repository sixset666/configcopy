Документ "Поступление дополнительных расходов"

Назначение

Документ "Поступление дополнительных расходов" предназначен для учета тех услуг сторонних организаций и собственной компании, которые оказывают влияние на себестоимость товаров. Например, в качестве такой услуги может быть, транспортировка или хранение товара. В документе можно указывать только услуги.

При проведении документ формирует хозяйственную операцию:

  • Поступление доп. расходов. Хозяйственная операция списывает дебиторскую задолженность поставщика по другим сделкам, если таковые были и начисляет кредиторскую задолженность компании перед поставщиком на регистре «Взаиморасчеты компании» на сумму предоставляемых услуг, приходует эту сумму на регистр «Партии товаров компании», распределяя её пропорционально товарам указанного документа «Поступление товаров» или на регистр «Доходы и расходы» (в зависимости от способа распределения дополнительных расходов каждой услуги). Также на регистре «Доходы и расходы» отражаются суммовые разницы.
  • Поступление доп. расходов внутреннее.  Хозяйственная операция списывает дебиторскую задолженность поставщика по другим сделкам, если таковые были и начисляет кредиторскую задолженность компании перед поставщиком на регистре «Взаиморасчеты компании» на сумму предоставляемых услуг, приходует эту сумму на регистр «Партии товаров компании», распределяя её пропорционально товарам указанного документа «Перемещение товаров» или на регистр «Доходы и расходы»  (в зависимости от способа распределения дополнительных расходов каждой услуги). Также на регистре «Доходы и расходы» отражаются суммовые разницы.

Относится к подсистеме: «Движения денежных средств».

Параметры

Длина номера: 10.

Периодичность номера: Год.

Тип номера: Строка.

Вводится на основании: «Поступление товаров», «Перемещение товаров», «Комплектация», «Разукомплектация».

Объекты, вводимые на основании: «Расходный кассовый ордер», «Счет фактура полученный», «Корректировка», «Событие».

Реквизиты

  • Дата входящего документа. Тип значения: «Дата». Дата входящего документа, используется для отражения даты бумажного документа, т.к. дата полученного документа может отличаться от даты регистрации документа в нашей системе.
  • Номер входящего документа. Тип значения: «Строка». Номер входящего документа, используется для отражения номера бумажного документа, т.к. номер полученного документа может отличаться от нашего внутреннего номера системы.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагенты». Контрагент, оказывающий услуги.
  • Документ основание. Реквизит является обязательным для заполнения. Ссылается на документы «Поступление товаров», «Перемещение товаров», «Комплектация», «Разукомплектация». Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов (по кнопке ). Допустимый тип документа-основания зависит от выбранной хозяйственной операции.
  • Договор. Реквизит является обязательным для заполнения. Ссылается на справочник «Договоры взаиморасчетов». Договор, по которому ведутся взаиморасчеты с контрагентом.
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ.
  • Регламентированный учет. Тип значения: «Булево». Флаг отражения документа в регламентированном учете. 
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ.
  • Тип Цен. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы цен». Тип цен, по которым были оказаны услуги.
  • Валюта. Реквизит является обязательным для заполнения. Ссылается на справочник «Валюты». Валюта документа.
  • Курс. Реквизит является обязательным для заполнения. Тип значения: «Строка». Курс валюты документа, на дату документа.  
  • Курс валюты упр. учета. Тип значения: «Строка». Курс валюты управленческого учета. (Возможность редактирования задается правом  "Редактирование курса управленческой валюты в документах").
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.  
  • Сумма. Тип значения: «Строка». Сумма документа.
  • Проект. Ссылается на справочник «Проекты». Проект, к которому относится документ.

Табличная часть «Товары»

  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Номенклатура (услуги).
  • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество единиц измерения номенклатуры.
  • Единица измерения. Реквизит является обязательным для заполнения. Ссылается на справочник «Единицы измерения». Единица измерения номенклатурной позиции.
  • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент пересчета по отношению к базовой единице номенклатуры.
  • Цена. Тип значения: «Число». Цена номенклатурной позиции.
  • Сумма. Тип значения: «Число». Сумма всегда без скидки. НДС входит в сумму, если это указано в типе цен. Сумма всегда равна Цена*Количество.
  • Всего. Тип значения: «Число». Итоговая сумма по строке документа со всеми налогами.
  • Ставка НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по номенклатурной позиции.
  • Сумма НДС. Тип значения: «Число». Сумма НДС по номенклатурной позиции.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора документов данного типа.
  • Форма документа. Основная форма документа. Используется для заполнения обязательных реквизитов.

Печатные формы

  • Поступление дополнительных расходов. Простая печатная форма, содержит список оказанных услуг поставщика.
  • ТОРГ-12. Унифицированная печатная форма №Торг-12, утвержденная постановлением Госкомстата России.
  • Печать этикеток и ценников. Обработка, позволяющая получить печатную форму этикеток и ценников заказанных товаров.

Способ вызова

Вызывается из меню "Документы --> Поступление ТМЦ--> Поступление дополнительных расходов".

Журналы, отображающие документ: «Поставщики».

Движения

  • Регистр сведений «Допроведение по партиям».
  • Остаточный регистр «Взаиморасчеты компании».
  • Остаточный регистр «Доходы и расходы».
  • Остаточный регистр «Партии товаров компании».

Последовательности, изменяемые документом: "Партии", "Взаиморасчеты".
