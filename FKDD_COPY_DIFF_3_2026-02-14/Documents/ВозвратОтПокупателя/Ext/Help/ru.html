Документ "Возврат от покупателя"

Назначение

Документ предназначен для отражения в оперативном учете операции по возврату от покупателей или комиссионеров ранее купленных товаров. В табличной части документа не могут фигурировать услуги, работы и т.п. При проведении документ пытается распределить товары по тем партиям, по которым они были отгружены, либо, если это невозможно - при отсутствии документа отгрузки, образует партию собственного товара.

При проведении документ формирует хозяйственные операции:

  • Возврат товаров от покупателя. Хозяйственная операция списывает дебеторскую задолженность покупателя по другим сделкам, если таковые были и начисляет кредиторскую задолженность компании перед покупателем или комиссионером на регистре «Взаиморасчеты компании», сторнирует материальные активы, на оптовых складах предприятия с собственным или комиссионным признаком партии, хранимые на регистрах «Остатки товаров компании» и «Партии товаров компании», сторнирует товарную задолженность перед комитентом по принятым на комиссию и проданным товарам на регистре «Реализованные товары», сторнирует сумму и себестоимость проданных товаров на регистре «Продажи», а так же списывает сумму продажи с регистра сведений «Накопление cумм».
  • Возврат товаров от покупателя (комиссия). Хозяйственная операция аналогична "Возврату товаров от покупателя" за тем исключением, что начисление кредиторской задолженности не происходит, а сторнируется товарная задолженность комиссионера перед нашей компанией с регистра «Партии товаров отданные», сторнирует товарную задолженность перед комитентом по принятым на комиссию и проданным товарам на регистре «Реализованные товары».

Относится к подсистеме: «Отгрузка ТМЦ».

Параметры

Длина номера: 10.

Периодичность номера: Год.

Тип номера: Строка.

Вводится на основании: «Реализация товаров», «Закрытие смены», «Инвентаризация товаров»,  «Ввод остатков товаров».

Объекты, вводимые на основании: «Возврат поставщику», «Перемещение товаров», «Расходный кассовый ордер», «Корректировка», «Счет фактура полученный», «Приходный складской ордер», «Платежное поручение».

Реквизиты

  • Склад компании. Реквизит является обязательным для заполнения. Ссылается на справочник «Склады (места хранения) компании». Склад компании, на который будет оприходован возвращенный товар.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагенты и контакты». Покупатель или комиссионер, возвращающий товар.
  • Договор. Реквизит является обязательным для заполнения. Ссылается на справочник «Договоры взаиморасчетов». Договор, по которому ведутся взаиморасчеты с контрагентом.
  • Документ отгрузки. Ссылается на документы «Ввод остатков товаров», «Закрытие смены», «Реализация товаров». Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов (по кнопке ).
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ.
  • Регламентированный учет. Тип значения: «Булево». Флаг отражения документа в регламентированном учете.
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ. 
  • Менеджер. Ссылается на справочник «Сотрудники». Сотрудник, заключивший сделку с контрагентом.
  • Тип Цен. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы цен». Тип цен, по которым возвращается товар.  
  • Валюта. Реквизит является обязательным для заполнения. Ссылается на справочник «Валюты». Валюта документа.
  • Курс. Реквизит является обязательным для заполнения. Тип значения: «Число». Курс валюты документа, на дату документа.
  • Курс валюты упр. учета. Тип значения: «Число». Курс валюты управленческого учета. (Возможность редактирования задается правом  "Редактирование курса управленческой валюты в документах"). 
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция. 
  • Время. Тип значения: «Дата». Время создания документа.
  • Сумма. Тип значения: «Число». Сумма документа.
  • Значение Скидки наценки. Ссылается на справочник «Типы скидок и наценок». Скидка/наценка, которая была применена при покупке товара.
  • Проект. Ссылается на справочник «Проекты». Проект, к которому относится документ.

Табличная часть «Товары»

  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Номенклатура (товар).
  • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество единиц измерения номенклатуры.
  • Единица. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Единицы измерения». Единица измерения номенклатурной позиции.
  • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент пересчета по отношению к базовой единице номенклатуры.
  • Количество базовое. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество номенклатуры в базовых единицах (коэффициент=1).
  • Цена. Тип значения: «Число». Цена номенклатурной позиции.
  • Процент наценки. Тип значения: «Число». Наценка на цену реализации для вычисления розничной цены.
  • Цена розничная. Тип значения: «Число». Розничная цена для оприходования номенклатуры на розничный склад.
  • Сумма. Тип значения: «Число». Сумма всегда без скидки. НДС входит в сумму, если это указано в типе цен. Сумма всегда равна Цена*Количество.
  • Всего. Тип значения: «Число». Итоговая сумма по строке документа со всеми налогами.
  • Сумма розничная. Тип значения: «Число». Розничная сумма для оприходования номенклатуры на розничный склад.
  • Процент скидки. Тип значения: «Число». Процент скидки.
  • Сумма скидки. Тип значения: «Число». Сумма скидки по строке документа.
  • Процент НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по номенклатурной позиции.
  • Сумма НДС. Тип значения: «Число». Сумма НДС по номенклатурной позиции.
  • Документ продажи. Реквизит является уникальным и обязательным для заполнения. Ссылается на документы «Реализация товаров», «Закрытие кассовой смены». Документ которым был продан товар.
  • Характеристика. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры, например, цвет, размер и т.п. 
  • Партия. Реквизит является уникальным. Ссылается на Характеристику «Типы Партий». Партия, на которую возвращаем товар.
  • Себестоимость. Тип значения: «Число». Себестоимость базовой единицы возвращенного товара.
  • Ячейка. Ссылается на справочник «Ячейки хранения». Ячейка хранения для номенклатуры. Значение по умолчанию хранится в номенклатуре в разрезе складов.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора документов данного типа.
  • Форма документа. Основная форма документа. Используется для заполнения обязательных реквизитов.

Печатные формы

  • Возврат от покупателя. Простая печатная форма, содержит список возвращаемых товаров от покупателя или комиссионера.
  • Печать этикеток и ценников. Обработка, позволяющая получить печатную форму этикеток и ценников товаров, у которых изменяют цены.

Способ вызова

Вызывается из меню "Документы --> Отгрузка ТМЦ --> Возврат от покупателя".

Журналы, отображающие документ: «Покупатели», «Складские документы».

Движения

  • Регистр сведений «Допроведение по партиям».
  • Остаточный регистр «Взаиморасчеты компании».
  • Остаточный регистр «Доходы и расходы».
  • Остаточный регистр «Остатки товаров компании».
  • Остаточный регистр «Партии товаров компании».
  • Остаточный регистр «Партии товаров отданные».
  • Оборотный  регистр «Продажи».
  • Остаточный регистр «Реализованные товары».

Последовательности, изменяемые документом: "Партии", "Взаиморасчеты".
