Документ "Электронное письмо"

Назначение

Документ предназначен для отправки исходящих электронных писем и регистрации входящих.

При проведении документ формирует хозяйственные операции:

  • Электронное письмо. Не создает движений.

Относится к подсистеме: «Сервис».

Параметры

Длина номера: 10.

Периодичность номера: Год.

Тип номера: Строка.

Вводится на основании: «Событие», «Контрагенты и контакты».

Объекты, вводимые на основании: «Событие», «Корректировка».

Реквизиты

  • Отправитель представление. Тип значения: «Строка». Представление отправителя.
  • Получатели представление. Тип значения: «Строка». Представление всех получателей письма Иванов Копии: Иванов, Сидоров Скрытые копии: Петров.
  • Копии. Тип значения: «Строка». Копии представления.
  • Скрытые копии. Тип значения: «Строка». Скрытые копии представления.
  • Тема. Реквизит является обязательным для заполнения. Тип значения: «Строка». Тема электронного сообщения.
  • Формат текста. Ссылается на перечисление «Формат текста». Вид текста почтового сообщения HTML, Простой текст.
  • Текст письма. Тип значения: «Строка». Текст электронного сообщения.
  • Почтовая папка. Реквизит является обязательным для заполнения. Ссылается на справочник «Группы писем электронной почты». Папка писем учетной записи, которой принадлежит документ.
  • Важность. Ссылается на перечисление «Важность». Важность электронного письма.
  • Учетная запись. Реквизит является обязательным для заполнения. Ссылается на справочник «Учетные записи электронной почты». Почтовый ящик, с которого было принято/отправлено сообщение .
  • Дата отправления. Тип значения: «Дата». Дата отправление электронного сообщения.
  • Документ основание. Ссылается на документ «Электронное письмо». Потомок электронного сообщения, документ основание для документа ответа или переадресованных документов.
  • Дата получения. Тип значения: «Дата». Дата получения электронного сообщения.
  • Уведомить о доставке. Тип значения: «Булево». Признак получения подтверждения доставки письма от почтового сервера.
  • Уведомить о прочтении. Тип значения: «Булево». Признак получения подтверждения прочтения от получателя письма.
  • Комментарий. Тип значения: «Строка». Комментарий документа.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Наименование фирмы, в которой создан документ.
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Пользователь создавший документ.
  • Статус письма. Ссылается на перечисление «Статусы сообщений». Статус письма Отправленное, Полученное, Сохраненное.
  • Наличие вложения. Тип значения: «Булево». Признак того, что к сообщению прикреплены файлы.
  • Идентификатор письма. Тип значения: «Строка». Идентификатор электронного сообщения, заданный почтовым сервером.
  • Не рассмотрено. Тип значения: «Булево». Признак того, что полученное сообщение не рассмотрено.
  • Ответ. Тип значения: «Булево». Признак того, что письмо является ответом.
  • Переадресация. Тип значения: «Булево». Признак того, что письмо сформировано в результате переадресации.

Табличная часть «Получатели письма»

  • Адрес электронной почты. Тип значения: «Строка». Электронный адрес получателя.
  • Представление. Тип значения: «Строка». Представление получателя сообщения.
  • Код группы адреса. Ссылается на перечисление «Код группы адреса электронного письма». Определяет группу адреса: кому, копии, скрытые копии.

Табличная часть «Прикрепленные файлы»

  • Файл. Тип значения: «Хранилище значения». Сохраненный файл.
  • Размер. Тип значения: «Число». Размер файла в байтах.

Формы

  • Форма документа. Основная форма объекта. Используется для заполнения обязательных реквизитов.
  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора документов данного типа.

Способ вызова

Вызывается из меню "Документы  --> Электронное письмо ".

    Дополнительная информация

    Статус письма изменяется пользователем косвенно, при совершении тех или иных действий (создание и сохранение письма, отправка, получение).

    Все входящие письма являются полученными. Статус «Сохраненное» письмо принимает, когда является так называемым «черновиком».

    Исходящее письмо – письмо к отправке, но по тем или иным причинам еще не отправленное на почтовый сервер (например, у пользователя нет прав на транспорт писем, нет связи с почтовым сервером). В состоянии «Исходящее» письмо еще может быть отредактировано.

    Все полученные письма недоступны для редактирования, кроме тех полей, которые не характеризуют электронное письмо как почтовое сообщение, а являются некими классификаторами данного письма в текущей конфигурации (например, группа писем, предмет и пр.).

    Для нового исходящего письма можно указать тип текста письма (простой текст, текст в формате HTML). Для текста в формате HTML в тело письма можно вставлять картинки.

    В полях «Кому», «Копии» и «Скрытые копии» пользователь может указать начало имени или адреса электронной почты адресата. После ввода более 2-х букв программа сама пытается отыскать по данным параметрам получателя. Если таковой найден, то подставляется его адрес (или имя и адрес, если были введены начальные буквы имени) ,  если же найдено несколько вариантов, то выводится список для выбора. Данная подстановка работает при включенной настройке "Использовать автоподбор адреса".

    Несколько адресов или имен получателей можно указывать через запятую.

    Для новых писем можно автоматически добавлять подпись в конец письма. Сама подпись заполняется в "Учетной записи" и добавляется в письмо в зависимости от настройки "Добавить подпись". С помощью кнопки "Подпись" на форме документа возможно ручное добавление подписи.

    На закладке "Дополнительно" можно посмотреть группу писем, к которой данное письмо принадлежит.
