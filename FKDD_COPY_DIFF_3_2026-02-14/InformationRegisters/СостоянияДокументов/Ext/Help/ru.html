Регистр сведений "Состояния документов"

Назначение

Регистр предназначен для хранения сведений об истории утверждения документов пользователями в разрезе пользователя, статуса документа, подразделения.

Относится к подсистеме: «Сервис».

Параметры

Периодичность:ПозицияРегистратора

Режим записи: ПодчинениеРегистратору

Измерения

  • Пользователь. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Пользователь, изменивший состояние документа.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение пользователя.
  • Статус документа. Реквизит является обязательным для заполнения. Ссылается на перечисление «Статусы документов». Статус документа после выполнения утверждения.

Ресурсы

  • Сумма. Тип значения: «Число». Краткое описание.

Реквизиты

  • Комментарий. Тип значения: «Строка». Произвольный комментарий.

Способ вызова

Вызывается из любого документа, для которого применяется утверждение, из нижней панели формы документа, подменю Утверждение - История.

Дополнительная информация

Информация в регистр записывается только при условии установки значения права "Использовать утверждение документов" в Истина. Данное право вызывается из пункта меню «Обработки»-«Рабочее место (АРМ)» . Также обязательными условиями являются установка значения права из групп настроек "Утверждение документов о пользователям", "Утверждение документов по подразделениям", соответствующего определенному документу, в значение Истина и указания списка возможных действий для указанного пользователя или подразделения (Подготовлен, Согласован, Утвержден, Отклонен).

Общие принципы использования утверждения документов следующие:

  • Права на утверждения (Подготовлен, Согласован, Утвержден, Отклонен), заданные через обработку «Обработки»-«Рабочее место (АРМ)» складываются друг другом. В итоге, например, если для подразделения Администрация будет стоять право "Утвержден", а для пользователя Администратор, относящегося к этому подразделению, только право "Подготовлен", то в результате суммирования этих прав пользователь Администратор будет иметь возможность подготовки и согласования определенного вида документов.
  • Выполнить любое из действий по утверждению можно как из формы документа, так и из формы списка документа. В случае, если документ еще не записан, будет выдан соответствующий вопрос о записи.
  • При создании нового документа, при использовании утверждения документов в целом для компании и использовании утверждения для данного документа и данного пользователя (подразделения) по умолчанию будет указан статус "Подготовлен", если у пользователя имеется право на это действие. Пользователь может  просто записать документ с присвоением ему статуса "Подготовлен" или выбрать необходимый статус.
  • В случае, если пользователь имеет право на выполнение предыдущего действия, то он может сразу выбрать необходимое ему действие. Например, для утверждения документа, если есть права на подготовку и согласование, будет достаточно сразу выбрать "Утвержден" из подменю.
  • При выполнении утверждения дополнительно можно настроить следующие права через обработку «Обработки»-«Рабочее место (АРМ)»:
  • "Отклонение при отмене проведения" - да/нет/спрашивать - присваивать документу статус "Отклонен" при выполнении отмены проведения документа.
  • "Автозаполнение статусов документов при включении утверждения" - да/нет/спрашивать - выполнять первичное заполнение для всех пользователей и подразделений для всех видов документов список прав значением "Подготовлен". Рекомендуется значение "Да".
  • "Отмена проведения при возврате к согласованию" - да/нет/спрашивать - выполнять отмену проведения документа при изменении статуса документа с "Утвержден" к "Согласован". Соответственно, право "Отклонение при отмене проведения" при этом не учитывается.
  • При использовании утверждения и выполнении проведения документа документу будет присвоен статус "Утвержден". Если у пользователя нет права на утверждение будет выдано сообщение о нарушении прав и документ не будет проведен.
  • Пользователь может выполнять "откат" статуса документа только если у него есть право на присвоение текущего статусу документа и если он является выполнившим данное присвоение статуса. Например, пользователь Иванченко может сделать "откат" к статусу "Согласован" документа "Авансовый отчет" только, если у него есть право на согласование и именно он выполнил это согласование.
