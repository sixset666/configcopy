Форма узла плана обмена "Удаленные подразделения"

Назначение

Форма предназначена для заполнения реквизитов распределенного узла информационной системы.

Реквизиты

  • Наименование. Наименование узла обмена.
  • Код. Код узла обмена. Не редактируется пользователем, в случае существования принятых сообщений обмена для данного узла.
  • Номер отправленного сообщения. Номер последнего отправленного сообщения обмена. Не редактируется пользователем. Устанавливается автоматически при удачной публикации сообщения обмена.
  • Номер принятого сообщения. Номер последнего принятого сообщения обмена. Не редактируется пользователем. Устанавливается автоматически при удачном получении сообщения обмена.
  • Подразделение компании. Ссылается на справочник «Подразделения компании». Ветка подразделений компании, обслуживаемая узлом базы данных и служащая фильтром при передаче изменений на уровне отдельных объектов.
  • Способ обмена. Определяет способ доставки/получения сообщений обмена. Возможные варианты значений реквизита:
     - файловый способ: при выборе данного варианта обмена, для доставки/получения сообщений обмена будут использованы ресурсы локальной сети.
     - MAPI: при выборе данного варианта обмена, для доставки/получения сообщений обмена будет использован почтовый клиент, установленный в системе по умолчанию.
     - SMTP/POP3: при выборе данного варианта обмена, для доставки/получения сообщений обмена будет использован прямой доступ к почтовым серверам.
     - FTP: при выборе данного варианта обмена, для доставки/получения сообщений обмена будет использованы ресурсы FTP сервера.
  • Каталог обмена. Путь к файлам обмена. Каталог сохранения/получения сообщений обмена. Используется при файловом способе обмена или использовании ресурсов FTP сервера.
  • Адрес электронной почты. Электронный адрес получателя сообщения обмена.
  • Сервер SMTP. Имя почтового сервера, используемого для отправки сообщений обмена.
  • Порт SMTP. Порт почтового сервера, используемого для отправки сообщений обмена.
  • Пользователь SMTP. Имя учетной записи, используемой для авторизации на сервере отправки сообщений обмена.
  • Пароль SMTP. Пароль учетной записи, используемой для авторизации на сервере отправки сообщений обмена.
  • Сервер POP3. Имя почтового сервера, используемого для приема сообщений обмена.
  • Порт POP3. Порт почтового сервера, используемого для приема сообщений обмена.
  • Пользователь POP3. Имя учетной записи, используемой для авторизации на сервере получения сообщений обмена.
  • Пароль POP3. Пароль учетной записи, используемой для авторизации на сервере получения сообщений обмена.
  • Профиль MAPI. Наименование почтового профиля, используемого в качестве почтового клиента для отправки/получения сообщений обмена. Если наименование профиля не задано, то будет использован почтовый профиль, установленный в системе по умолчанию.
  • Пароль MAPI. Пароль для доступа к почтовому профилю.
  • Сервер FTP. Адрес общедоступного FTP сервера, используемого для сохранения/получения сообщений обмена.
  • Порт FTP. Порт FTP сервера, используемого для сохранения/получения сообщений обмена.
  • Пользователь FTP. Имя учетной записи, используемой для авторизации на FTP сервере.
  • Пароль FTP. Пароль учетной записи, используемой для авторизации на FTP сервере.

Дополнительная информация

Проверку корректности введеных данных и возможности сохранения/получения сообщений обмена, можно произвести с помощью кнопки "Проверить", после нажатия которой, будет сформирован тестовый файл и в зависимости от установленного способа обмена, будет предпринята попытка публикации и получения тестового файла.
