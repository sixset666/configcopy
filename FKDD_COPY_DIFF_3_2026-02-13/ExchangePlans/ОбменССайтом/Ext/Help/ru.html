План обмена "Обмен с сайтом"

Назначение

Предназначен для обмена данными между информационными базами 1С:Предприятия 8.1 и веб-ориентированными системами управления содержанием "1С Битрикс". Обмен производится данными о свойствах, характеристиках, стоимости, количестве номенклатуры, а также данными о заказах покупателей.

План обмена содержит информацию о сайтах, которые могут обмениваться с информационнной базой 1С:Предриятие 8.1, определяет ручной или автоматический режим обмена. Ручной обмен данными осуществляется посредством вызова обработки "обмен с сайтом", автоматический обмен реализован с помощью фоновых заданий, управление которыми производится из "АРМ Администратора". Для каждого сайта можно задать его префикс, адрес в сети интернет и необходимый перечень реквизитов, описывающих сайт.

Механизм обмена информацией с сайтом основан на передаче и приеме xml файлов в формате, соответствующем спецификации CommerceML 2.04. Данное обстоятельство позволяет реализовывать обмен не только с системой управления содержанием "1С Битрикс", но и с другими интернет системами, поддерживающими обмен в формате CommerceML. Передача xml файлов на сайт осуществляется по протоколу http через post запрос. При этом файлы предварительно фрагментируются в соответствии с настройками, указанными на сайте в зависимости от объема данных, и могут архивироваться в формате zip для оптимизации скорости передачи данных.

Относится к подсистеме: «Сервис».

Параметры

Длина кода: 3.

Длина наименования: 80.

Реквизиты

  • Обмен товарами Тип значения: «Булево». Признак необходимости выгрузки номенклатуры.
  • Обмен заказами Тип значения: «Булево». Признак необходимости выгрузки\загрузки заказов покупателя.
  • Выгружать картинки Тип значения: «Булево». Признак необходимости выгрузки картинок номенклатуры на сайт.
  • Выгружать только изменения Тип значения: «Булево». Признак необходимости выгрузки изменений данных, иначе признак выгрузки всех данных.
  • Выгружать на сайт Тип значения: «Булево». Признак необходимости выгрузки на сайт, иначе выгрузка будет осуществляться в каталог.
  • Каталог обмена HTTP. Реквизит является обязательным для заполнения. Тип значения: «Строка». Файловый путь к каталогу куда будут выгружаться файлы необходимые при обмене.
  • Сервер HTTP. Реквизит является обязательным для заполнения. Тип значения: «Строка». URL адрес скрипта, производящего обмен данными со стороны сайта.
  • Порт HTTP. Реквизит является обязательным для заполнения. Тип значения: «Строка». Порт http сервера, на котором расположен сайт.
  • Пользователь HTTP. Реквизит является обязательным для заполнения. Тип значения: «Строка». Учетная запись на сайте, под которой производится обмен данными.
  • Пароль HTTP. Реквизит является обязательным для заполнения. Тип значения: «Строка». Пароль учетной записи, под которой происходит обмен данными.
  • Сервер прокси HTTP Тип значения: «Строка». URL адрес скрипта, доступ к которому осуществляется через http прокси.
  • Порт прокси HTTP Тип значения: «Строка». Порт http прокси, через который осуществляется доступ к web серверу.
  • Пользователь прокси HTTP Тип значения: «Строка». Имя учетной записи http прокси.
  • Пароль прокси HTTP Тип значения: «Строка». Пароль учетной записи http прокси.
  • Группа для новых контрагентов Ссылается на группу справочника "Контрагенты и контакты". Элемент родитель создаваемых контрагентов.
  • Таблица замены представлений валют Таблица значений, содержащая валюту и её название которым будет заменено её наименование при выгрузке. Так как при выгрузке валюты с наименованием на русском языке на строне вервера происходит ошибка, то наименование необходимо заменять на наименование на английском (например, "руб." -> "RUB").
  • Подразделение Ссылается на справочник "Подразделения компании". Данный параметр используется при определении цены товара, который выгружается в пакет предложений.
  • Определять цены контрагента Является переключателем для задания способа определения цены в контексте контрагентов.
  • Организация Ссылается на элементы справочника "Организации". Организация, которая подставляется в соответствующий реквизит документа ЗаказПокупателя.
  • Менеджер Ссылается на элементы справочника "Пользователи". Менеджер, осуществивший регистрацию заказа пользователя на сайте.
  • Подразделение Ссылается на элементы справочника "Подразделения компании". Подразделение, которое подставляется в создаваемый заказ покупателя.
  • Тип цены Ссылается на элементы справочника "Типы цен". Тип цены, который подставляется в соответствующий реквизит документа Заказ покупателя.
  • Записывать документы текущей датой Тип значения: «Булево». Признак записи документа текущей датой, в противном случае используется дата загружаемого документа.
  • Проводить документы Тип значения: «Булево». Признак проведения загружаемых документов в противном случае документы только записываются.
  • Проводить документы оперативно Тип значения: «Булево». Признак оперативного проведения создаваемых документов.
  • Способ идентификации контрагентов Тип значения: «Строка». Признак идентификации контрагентов либо по ИНН, либо по наименованию.
  • Сохраненные настройки построителя Тип значения: «ХранилищеЗначения». Настройки отбора построителя отчета, который используется при отборе выгружаемой номенклатуры.
  • Вид плательщика Ссылается на перечисление "Виды контрагентов". Вид плательщика, используемый при обмене заказами.
  • Регламентное задание Тип значения: «Строка». Регламентное задание.
  • Использовать регламентные задания Тип значения: «Булево». Признак использования регламентных заданий.
  • Комментарий Тип значения: «Строка». Комментарий к сайту плана обмена.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора сайтов обмена.
  • Форма узла. Форма, используемая для заполнения обязательных реквизитов сайта обмена.
  • Диалог регламентного задания. Форма, используемая для настройки регламентного задания обмена, осуществляемого в фоновом режиме.

Способ вызова

Вызывается из меню  "Справочники--> Структура компании -->Обмен с сайтом".
