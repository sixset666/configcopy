Дата документа - дата и время документа (момент документа). Используется для расчета (см ниже формулы) и для заполнения номенклатуры.

Вначале пользователь заполняет номенклатуру через меню "подбор" и "заполнение".

Кроме универсального "подбор номенклатуры" есть заполнение номенклатуры

-«Заполнение группой номенклатуры (остатки на дату)» - все запчасти указанной группы, которые имеют остатки на складах организации документа на момент документа (поэтому на момент заполнения документ должен быть записан);

-«Заполнение группой номенклатуры (имеющей мин. остатки)» - все запчасти указанной группы, для которых установлен минимальный остаток по подразделению документа на момент документа (поэтому на момент заполнения документ должен быть записан). Если нужно переустановить минимальные остатки, уже установленные, можно пользоваться этой командой.

-«Заполнение группой номенклатуры (вся группа)» -это вообще все элементы группы. Если нужно устанавливать минимальные остатки по всем позициям справочника – нужно пользоваться этой командой.

-по производителю, это позиции номенклатуры, имеющие остатки (даже зарезервированные) на момент документа по всем складам данного производителя из СпискаПараметров_ТТ с кодом 000015015.

После заполнения номенклатуры нужно выбрать "расчет"-"расчет минимального остатка"

Формулы для меню "Расчет минимального остатка":

Расход товара (количество) за предыдущие 12 месяцев до месяца даты документа = ОбщийРасход

СреднедневнойРасхода = (ОбщийРасход/365)

СреднедневнойРасход*ПлечоДоставки= МинимальныйОстаток1Уровня, ПлечоДоставки программно установлено = 14 (либо его можно задать в "параметрах" для номенклатуры: регистр сведений "Дополнительные параметры для расчета минимальных остатков")

СреднедневнойРасход*(ПлечоДоставки+СтраховойЗапас) = МинимальныйОстаток2Уровня, СтраховойЗапас программно установлено =14 (либо его можно задать в "параметрах" для номенклатуры: регистр сведений "Дополнительные параметры для расчета минимальных остатков")

Для определения Группа_АВС используется алгоритм:

-если число месяцев рассматриваемого года до наступления месяца документа, где расход номенклатуры (продажа) >0, больше КоличестваМесяцевГруппыА, то это товар группы А.

По умолчанию программно КоличествоМесяцевГруппыА = 6, соответственно товар с продажами в 7-12 месяцах года относится к группе А.

В регистре сведений "Дополнительные параметры для расчета минимальных остатков" можно изменить данный параметр для позиции номенклатуры.

-если число месяцев рассматриваемого года до наступления месяца данного документа, где расход номенклатуры (ее продажа) >0, больше КоличестваМесяцевГруппыВ, то это товар группы В.

По умолчанию программно задано КоличествоМесяцевГруппыВ = 2. Т.е. товары с продажами больше 2, до 6 месяцев - это товары группы B.

В регистре сведений "Дополнительные параметры для расчета минимальных остатков" можно изменить данный параметр для позиции номенклатуры.

-иначе товар определяется как С. Это от 0 до 2 месяцев, имеющих продажи, за год.

Для определения цены за указанный период "сумма продажи товара за год по номенклатуре" делится на общее количество проданного товара, средняя цена.

Из средней цены определяется Тип_XYZ:

-если 0 <= цена <=2000, это X

-если 2000<цена<=10000, это Y

-иначе Z, т.е. цена>10000.

Для определения МинимальногоОстатка3Уровня используется схема:

Если Группа_ABC = А то:

-МинимальныйОстаток3Уровня =МинимальныйОстаток2Уровня

Если Группа_ABC = В то:

-если Тип_XYZ = X то:

МинимальныйОстаток3Уровня = МинимальныйОстаток2Уровня

-если Тип_XYZ =  Y то:

МинимальныйОстаток3Уровня = МинимальныйОстаток1Уровня

-если Тип_XYZ =  Z то:

МинимальныйОстаток3Уровня = 1

Если Группа_ABC = C то:

-если Тип_XYZ = X то:

МинимальныйОстаток3Уровня = МинимальныйОстаток1Уровня

-если Тип_XYZ =  Y то:

МинимальныйОстаток3Уровня = 1

-если Тип_XYZ =  Z то:

МинимальныйОстаток3Уровня = 0

РассчитанныйМинимальныйОстаток = Округленный МинимальныйОстаток3Уровня в сторону БОЛЬШЕГО ЦЕЛОГО ЧИСЛА

Колонка УстанавливаемыйМинимальныйОстаток заполняется равной Рассчитанному.

При необходимости УстанавливаемыйМинимальныйОстаток редактируется. Имеющие разницу строки выделяются шрифтом. Обязателен комментарий-причина изменения минимального остатка. 

При проведении на момент времени документа создаются записи регистра сведений "дополнительные реквизиты номенклатуры", значение "минимальный остаток" по основному подразделению. При отмене проведения записи на момент времени документа по позициям номенклатуры удалятся

Если нужно будет перерассчитать заполненный документ нужно пользоваться "расчет"-"расчет без удаления установленных значений" - колонки "устанавливаемый" и "комментарий" не будут затираться.
