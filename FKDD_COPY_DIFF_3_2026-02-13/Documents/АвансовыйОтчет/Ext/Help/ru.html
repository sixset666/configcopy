Документ "Авансовый отчет"

Назначение

Авансовый отчет предназначен для списания сумм задолженностей с подотчетного лица, с отнесением их на выбранную статью расходов. Кроме того, взаиморасчеты с подотчетника могут списываться на взаиморасчеты с другим подотчетным лицом или поставщиком, закрывая, таким образом, кредиторскую задолженность. 

При проведении документ формирует хозяйственную операцию:

  • Авансовый отчет. Хозяйственная операция списывает кредиторскую задолженность подотчетного лица перед компанией на регистре «Взаиморасчеты компании», в зависимости от значения флага "На взаиморасчеты" или "На доходы и расходы" делает разные движения. При значении "На взаиморасчеты" списывает кредиторскую задолженность перед поставщиком по регистру «Взаиморасчеты компании», а при значении "На доходы и расходы" списывает задолженность подотчетника на выбранную статью расхода по регистру «Доходы и расходы» компании.

Относится к подсистеме: «Взаиморасчеты».

Параметры

Длина номера: 10, тип номера: Строка.

Периодичность номера: Год

Вводится на основании: «Расходный кассовый ордер», «Банковская выписка».

Объекты, вводимые на основании: «Приходный кассовый ордер», «Расходный кассовый ордер», «Корректировка», «Приходный складской ордер».

Реквизиты

  • Склад. Реквизит является обязательным для заполнения при наличии строк в табличной части "Товары". Ссылается на справочник «Склады (места хранения) компании». Склад компании, на который приходуется товар.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагенты и контакты». Подотчетное лицо.
  • Договор взаиморасчетов. Реквизит является обязательным для заполнения. Ссылается на справочник «Договоры взаиморасчетов». Договор, по которому ведутся взаиморасчеты с подотчетником.
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу.
  • Организация. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ.
  • Подразделение. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ.
  • Автор. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ.
  • Менеджер. Ссылается на справочник «Сотрудники». Сотрудник, заключивший сделку с контрагентом.
  • Документ-основание. Ссылается на документ  «Расходный кассовый ордер». Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов (по кнопке ).
  • Регламентированный учет. Тип значения: «Булево». Флаг отражения документа в регламентированном учете.
  • Управленческий учет. Тип значения: «Булево». Флаг отражения документа в управленческом учете.
  • Тип Цен. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы цен». Тип цен, в которых отгружается товар.
  • Валюта. Ссылается на справочник «Валюты». Валюта документа.
  • Курс. Тип значения: «Число». Курс валюты документа на дату документа.
  • Курс валюты управленческого учета. Тип значения: «Число». Курс валюты управленческого учета на дату документа. (Возможность редактирования задается правом пользователя "Редактирование курса управленческой валюты в документах").
  • Сумма. Тип значения: «Число». Сумма документа.
  • Операция. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.
  • Период с. Тип значения: «Дата». Начало периода для поиска выданных подотчетнику авансов (заполнение табличной части "Выданные авансы").
  • Период по. Тип значения: «Дата». Конец периода для поиска выданных подотчетнику авансов (заполнение табличной части "Выданные авансы").
  • Проект. Ссылается на справочник «Проекты». Проект, к которому относится документ.

Табличная часть «Выданные авансы»

  • Документ-основание. Реквизит является обязательным для заполнения. Ссылается на документ «Расходный кассовый ордер» или «Банковская выписка», . Документ, по которому выданы денежные средства подотчетнику.
  • Сумма. Реквизит является обязательным для заполнения. Тип значения: «Число». Сумма выданных денежных средств в валюте взаиморасчетов с подотчетником.

Табличная часть «Расходы»

  • Дата расхода. Тип значения: «Дата». Когда был произведен расход денежных средств.
  • Содержание. Тип значения: «Строка». Описание произведенного расхода с содержанием назначения этого расхода.
  • Документ расхода. Тип значения: «Строка». Текстовая информация о документе, подтверждающем расход денежных средств, приложенный к авансовому отчету.
  • Статья расхода. Ссылается на справочник «Статьи доходов и расходов». Статья, на которую списываются расходы с взаиморасчетов подотчетного лица, в том случае, если флажок На расчеты установлен в значение Ложь.
  • Сумма. Тип значения: «Число». Сумма, на которую произведен расход (например, закупленные товары).
  • Договор. Ссылается на справочник «Договоры взаиморасчетов». Договор взаиморасчетов с контрагентом, с которым подотчетник произвел расчеты.
  • На расчеты. Тип значения: «Булево». Истина - отнести сумму на взаиморасчеты с контрагентом, Ложь - отнести сумму на доходы и расходы.
  • Расчеты с. Ссылается на справочник «Контрагенты и контакты». Поставщик, расчеты с которым происходят через подотчетное лицо, если флажок На расчеты установлен в Истину.
  • Договор. Ссылается на справочник «Договоры взаиморасчетов». Договор взаиморасчетов с этим поставщиком.
  • Документ-основание. Ссылается на документ «Авансовый отчет», «Акт приема-передачи ценных бумаг», «Банковская выписка», «Ввод остатков взаиморасчетов», «Взаиморасчет», «Возврат от покупателя», «Заказ внутренний», «Заказ поставщику», «Корректировка долга», «Начисление зарплаты», «Обслуживание актива», «Отчет комитенту», «Переоценка валютных средств», «Поступление дополнительных расходов», «Поступление товаров», «Приходный кассовый ордер», «Чек на оплату». Документ, который мы закрываем, в случае если флажок На расчеты установлен в Истину. Этот документ должен быть от выбранного поставщика.

Табличная часть «Товары»

  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Товар, приобретенный подотчетным сотрудником.
  • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество единиц измерения номенклатуры.
  • Единица измерения. Реквизит является обязательным для заполнения. Ссылается на справочник «Единицы измерения». Единица измерения номенклатурной позиции.
  • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент пересчета по отношению к базовой единице номенклатуры.
  • Количество базовое. Тип значения: «Число». Количество номенклатуры в базовых единицах (коэффициент=1).
  • Цена. Тип значения: «Число». Цена номенклатурной позиции.
  • Сумма. Тип значения: «Число». Сумма всегда без скидки. НДС входит в сумму, если это указано в типе цен. Сумма всегда равна Цена*Количество.
  • Всего. Тип значения: «Число». Итоговая сумма по строке документа со всеми налогами.
  • Процент НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по номенклатурной позиции.
  • Сумма НДС. Тип значения: «Число». Сумма НДС по номенклатурной позиции.
  • ГТД / РНПТ. Ссылается на справочник «ГТД». Номер грузовой таможенной декларации (для автомобилей поступивших по импорту) или регистрационный номер (РНПТ) для прослеживаемых товаров.
  • Характеристика. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры, например, цвет, размер и т.п.
  • Процент наценки. Тип значения: «Число». Наценка на закупочную цену для вычисления розничной цены.
  • Цена розничная. Тип значения: «Число». Розничная цена для оприходования номенклатуры на розничный склад.
  • Сумма розничная. Тип значения: «Число». Розничная сумма для оприходования номенклатуры на розничный склад.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора документов данного типа.
  • Форма документа. Основная форма документа. Используется для заполнения обязательных реквизитов.

Печатные формы

  • Авансовый отчет (АО-1). Печатная унифицированная форма № АО-1.
  • Авансовый отчет. Простая печатная форма, выводящая на печать основные реквизиты документа.

Способ вызова

Вызывается из меню "Документы --> Взаиморасчеты --> Авансовый отчет".

Движения

  • Остаточный регистр «Взаиморасчеты компании».
  • Остаточный регистр «Доходы и расходы». 
  • Остаточный регистр «ГТД партий товаров компании». 
  • Остаточный регистр «Остатки товаров компании». 
  • Остаточный регистр «Партии товаров компании».
  • Регистр сведений «Допроведение по партиям». 
  • Регистр сведений «Состояния документов». 
  • Регистр сведений «Цены». 

Последовательности, изменяемые документом: "Взаиморасчеты".
