Документ "Чек"

Назначение

Документ "Чек" предназначен для оформления массовой продажи товаров населению в розницу. Предполагается, что продажа товара производится с розничного склада и расчет с клиентом производится немедленно и наличными деньгами или картой.

В документе "Чек" можно указать склад, с которого производится отгрузка товаров. Таким образом, можно оформлять с помощью одной ККМ розничную продажу с разных складов (например, с помощью одной ККМ можно осуществлять продажу в разных секциях магазина).

Как в любом другом документе, в документе "Чек" может быть установлена скидка по каждой позиции номенклатуры.
Документы вида "Чек"  оформляются в конфигурации в течение кассовой смены ККМ.
По окончании кассовой смены ККМ специальным регламентным документом «Закрытие смены» все текущие документы "Чек" удаляются; вместо них формируется и проводится обобщающий документ «Закрытие смены», который подводит итоги продажи в розничной торговле конкретной ККМ.

При проведении документ формирует хозяйственные операции:

  • Чек. Хозяйственная операция списывает товары с розничного склада по регистру «Остатки товаров компании» и приходует розничную выручку за товар на регистры «Кассы ККМ» и «Доходы и расходы». А так же регистрирует сумму продажи на регистре сведений «Накопление cумм». 
  • Чек на возврат. Хозяйственная операция полностью обратная операции "Чек".
  • Отложенный чек. Не создает движений. 

Относится к подсистеме: «Отгрузка ТМЦ».

Параметры

Длина номера: 10.

Периодичность номера: Год.

Тип номера: Строка.

Вводится на основании: «Чек».

Объекты, вводимые на основании: «Чек», «Корректировка», «Счет фактура выданный», «Событие».

Реквизиты

  • Склад компании (отдел). Реквизит является обязательным для заполнения. Ссылается на справочник «Склады (места хранения) компании». Розничный склад компании, с которого продаем товар.
  • Касса ККМ. Реквизит является обязательным для заполнения. Ссылается на справочник «Кассы ККМ». Касса ККМ, на которой пробивается чек.
  • Контрагент. Ссылается на справочник «Контрагенты и контакты». Покупатель, которому мы продали товар.
  • Карточка. Ссылается на справочник «Карточки». Дисконтная карта покупателя, по которой продавали товар.
  • ФР. Фискальный регистратор, на котором был пробит чек.
  • Номер чека. Тип значения: «Число». Фискальный номер чека пробитого на ФР.
  • Дата ФР. Тип значения: «Дата». Дата пробития чека на ФР.
  • Номер документа. Тип значения: «Число». Сквозной номер документа ФР.
  • Номер смены. Тип значения: «Число». Номер смены ФР.
  • Платежная карта. Ссылается на справочник «Типы платежных карт». Тип платежной карты, по которой была произведена безналичная оплата (American Express, Cirrus/Maestro, Eurocard/Mastercard, Sbercard).
  • Наличными. Тип значения: «Число». Сумма, полученная наличными.
  • Безналичными. Тип значения: «Число». Сумма, полученная безналичными с использованием платежной карты.
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в которой был выписан документ.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица от которого выписан документ.
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ.  
  • Документ основание. Ссылается на документ «Чек». Документ, на основании которого введен данный, необходим для привязки подчиненных документов к документу основанию, с помощью этого реквизита, затем строится дерево подчиненных документов (по кнопке ).
  • Тип цен. Реквизит является обязательным для заполнения. Ссылается на справочник «Типы цен». Тип Цен, в которым продавали товар, по умолчанию розничная.
  • Валюта. Реквизит является обязательным для заполнения. Ссылается на справочник «Валюты». Валюта документа, всегда совпадает с валютой регламентированного учета.
  • Курс. Реквизит является обязательным для заполнения. Тип значения: «Число». Курс валюты документа на дату документа.
  • Курс валюты управленческого учета. Тип значения: «Число». Курс валюты управленческого учета. (Возможность редактирования задается правом  "Редактирование курса управленческой валюты в документах").
  • Вид оплаты. Ссылается на перечисление «Виды оплаты». Вид оплаты покупателем. Значение реквизита устанавливается автоматически в зависимости от сумм наличных и безналичных оплат документа покупателем.
  • Скидка/наценка. Ссылается на справочник «Типы скидок и наценок». Скидка/наценка на документ в целом. Может быть подставлена автоматически (для автоматических скидок) или выбрана пользователем (для ручных скидок).
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.
  • Сумма документа. Тип значения: «Число». Сумма в валюте документа, налоги включены согласно типу цен документа.

Табличная часть «Товары»

  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Номенклатура (товар).
  • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество единиц измерения номенклатуры.
  • Остаток. Тип значения: «Число». Количество номенклатуры, имеющейся на складе компании.
  • Единица. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочники «Единицы измерения». Единица измерения номенклатурной позиции.
  • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент пересчета по отношению к базовой единице измерения номенклатуры.
  • Цена. Тип значения: «Число». Цена номенклатурной позиции.
  • Сумма. Тип значения: «Число». Сумма всегда без скидки. НДС входит в сумму, если это указано в типе цен. Сумма всегда равна Цена*Количество.
  • Отдел. Реквизит является обязательным для заполнения. Ссылается на справочник «Склады (места хранения) компании». Место хранения товаров.
  • Процент скидки. Тип значения: «Число». Процент скидки на продаваемый товар.
  • Скидка. Тип значения: «Число». Сумма скидки по строке документа.
  • Процент скидки строки. Тип значения: «Число». Процент скидки на строку документа.
  • Скидка на товар. Тип значения: «Число». Сумма скидки на конкретный товар.
  • Сумма скидки строки. Тип значения: «Число». Сумма скидки строки документа.
  • Всего. Тип значения: «Число». Сумма всего.
  • Процент НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по номенклатурной позиции.
  • Сумма НДС. Тип значения: «Число». Сумма НДС по номенклатурной позиции.
  • Характеристика номенклатуры. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика/серия номенклатуры, например, цвет, размер и т.п.
  • Набор. Реквизит является уникальным. Ссылается на справочник «Номенклатура». Набор, которому принадлежит текущий товар.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора документов данного типа.
  • Форма документа. Основная форма объекта. Используется для заполнения обязательных реквизитов.

Печатные формы

  • ТОРГ-12 (Товарная накладная за поставщика). Унифицированная печатная форма №Торг-12, утвержденная постановлением Госкомстата России.
  • Чек. Простая печатная форма чека.
  • Печать этикеток и ценников. Обработка, позволяющая получить печатную форму этикеток и ценников проданных товаров.

Способ вызова

Вызывается из меню "Документы --> Отгрузка ТМЦ --> Чек"

Движения

  • Остаточный регистр «Взаиморасчеты компании».
  • Остаточный регистр «Доходы и расходы».
  • Остаточный регистр «Остатки товаров компании».
  • Остаточный регистр «Кассы ККМ».
