Документ "Тест-драйв"

Назначение

Используется для отражения в системе информации о проведение тест-драйвов автомобилей. Документ формирует записи в регистре "График работы ресурсов" о загруженности автомобилей и сотрудников.

 

На основании тест-драйва могут быть введены документы «Опрос». На форме документа в разделе "Результаты анкетирования" присутствуют гиперссылки для быстрого перехода к предварительному и итоговому анкетному опросу. Ввод нового предварительного и итогового опроса доступен для автомобилей, для которых в регистре «Настройки тест-драйва» указаны соответствующие анкеты. Если опросы уже введены, гиперссылки позволяют открыть формы соответствующих документов.

 

Список хозяйственных операций:

  • Тест-драйв. 

Относится к подсистеме: «Автосалон».

Параметры

Длина номера: 10

Периодичность номера: Год

Тип номера: Строка

Вводится на основании: «Событие», «Заказ на автомобиль».

Объекты, вводимые на основании: «Опрос».

Реквизиты

  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ.
  • Менеджер. Реквизит является обязательным для заполнения. Ссылается на справочник «Сотрудники». Сотрудник, ответственный за проведение тест-драйва.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании в котором был выписан документ.
  • Комментарий. Произвольный комментарий к документу.
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция.
  • Автомобиль. Реквизит является обязательным для заполнения. Ссылается на справочник «Автомобили». Автомобиль для проведения тест-драйва.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагенты». Клиент, для которого проводится тест-драйв.
  • Телефон. Телефон клиента, по которому он обратился в автосалон. Может быть выбран из списка, отредактирован или введен новый. Редактируется в диалоге. Для нового номера по умолчанию подставляется вид контактной информации "мобильный телефон". Если телефон с данным видом контактной информации уже задан в регистре для данного клиента, то задается вопрос о замене телефона либо выборе другого вида контактной информации для записи.
  • Дата начала. Реквизит является обязательным для заполнения. Дата начала тест-драйва.
  • Дата окончания. Реквизит является обязательным для заполнения. Дата окончания тест-драйва.
  • Контактный телефон. Поле для указания телефона клиента. Редактирование телефона осуществляется в отдельной форме.
  • Состояние. Реквизит является обязательным для заполнения. Ссылается на перечисление «СостоянияСобытий». Текущее состояние выполнения тест-драйва.
  • Маршрут. Реквизит является обязательным для заполнения. Ссылается на справочник «Маршруты тест-драйвов». Маршрут тест-драйва.
  • Пробег на начало. Пробег на одометре автомобиля на момент начала тест-драйва.
  • Пробег на конец. Пробег на одометре автомобиля на момент окончания тест-драйва.
  • Результат тест-драйва. Ссылается на справочник «Результаты контакта». Результат проведения тест-драйва.

Формы

  • Форма списка. Основная форма списка. Используется для отображения списка документов данного типа.
  • Форма документа. Основная форма документа. Используется для ввода и редактирования документов данного типа.
  • Форма ввода дат документа. Форма используется для ввода дат начала и окончания тест-драйва. Содержит раздельный ввод даты и времени соответствующих реквизитов.

Печатные формы

  • Бланк тест-драйва.

Способ вызова

Вызывается из меню "Документы --> Автосалон--> Тест-драйв".

Дополнительная информация

Документ может находится в различных состояниях:

1. При проведении документа со статусом "Запланировано" формируется напоминание менеджеру, указанному в документе,  о предстоящем тест-драйве за N часов до тест-драйва, где N задается настройкой "Срок напоминания перед тест драйвом" для подразделения. Если в настройке задан нулевой срок, то напоминания не формируются.

2. По результатам прозвона клиенту документ переводится в состояние "Утверждено".

3. В момент начала тест-драйва документ переводится в состояние "Выполняется". Перевод в состояние "Выполняется" допустим только после заполнения предварительной анкеты, если такая задана для данной модели автомобиля.

4. После окончания тест-драйва документ переводится в состояние "Завершено". Перевод в состояние "Завершено" допустим только после заполнения итоговой анкеты, если такая задана для данной модели автомобиля.

5. Документ переводится в состояние "Отменено", если тест-драйв по каким-либо причинам был отменен. Если клиент приехал не вовремя (раньше или позже), то текущий документ переводится в состояние "Отменено" и вводится новый документ.
