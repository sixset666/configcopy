Документ "Бюджет продаж"

Назначение

Документ "Бюджет продаж" предназначен для планирования продаж номенклатуры или категорий номенклатуры за выбранный период времени, в определенном подразделении и для выбранного сценария планирования, который определяет периодичность. Документ позволяет запланировать данные, как по категориям номенклатуры, так и по номенклатуре.  Вид планируемой аналитики определяется хоз. операцией, т.е. выбранным видом бюджета. В зависимости от выбранного вида бюджета табличная часть документа именуется либо "Товары" либо "Типы номенклатуры". Запланированные документом данные продаж относятся к выбранному периоду планирования, который соответствует периодичности сценария планирования (поддерживаются следующие периоды планирования: месяц, квартал, год). Дата документа может не совпадать с периодом планирования, но не может быть больше периода планирования.

Возможность ввода "Бюджета продаж" для того или иного подразделения определяется и контролируется настройкой регистра сведений «Параметры планирования».

Суммы продаж планируются в валютах управленческого и регламентированного учета, пересчет из одной валюты в другую осуществляет по курсу на дату документа.

Документ позволяет производить ручное заполнение, т.е непосредственное добавление элементов в табличную часть и автоматическое заполнение, на основании фактических или плановых данных продаж предыдущих и текущего периода, с использованием методов математического прогнозирования или на основании другого бюджета продаж. Автоматическое заполнение реализовано в виде набора дополнительных форм документа.

 

При проведении документ формирует хозяйственные операции:

  • Бюджет продаж по категориям. Формируются движения по регистру накопления: «Бюджет продаж», в котором производится накопление запланированных данных продаж. Измерение "Номенклатура" заполняется позициями из табличной части Типы номенклатуры.
  • Бюджет продаж по номенклатуре. Формируются движения по регистру накопления: «Бюджет продаж», в котором производится накопление запланированных данных продаж.  Измерение "Номенклатура" заполняется позициями  из табличной части Товары. 

Относится к подсистеме: «Планирование».

Параметры

Длина номера: 10.

Периодичность номера: Год.

Тип номера: Строка.

Вводится на основании: «Бюджет продаж».

Объекты, вводимые на основании: «Бюджет продаж», «Бюджет закупок», «Корректировка».

Реквизиты

  • Сценарий планирования.  Реквизит является обязательным для заполнения. Ссылается на справочник «Сценарии планирования». Определяет периодичность планирования.
  • Дата планирования. Реквизит является обязательным для заполнения. Тип значения: «Дата». Начальная дата периода, на который происходит планирование. Зависит от реквизита "Периодичность" выбранного сценария планирования. 
  • Планирование себестоимости. Тип значения: «Булево». Признак планирования себестоимости в документе. Устанавливается при усеченной схеме операционных бюджетов, когда не составляется отдельный бюджет закупок.
  • Комментарий. Тип значения: «Строка». Произвольный комментарий к документу. 
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, в котором был выписан документ. 
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Идентификатор юридического лица, от которого выписан документ.
  • Автор. Реквизит является обязательным для заполнения. Ссылается на справочник «Пользователи». Автор документа - пользователь, породивший документ.
  • Документ основание.  Ссылается на документ «Бюджет продаж». Содержит документ-основание, в случае, если применялось автоматическое заполнение по документу-основанию.
  • Тип анализа. Ссылка на перечисление «Типы анализа». Используется при автоматическом заполнении. Определяет тип анализируемых данных. 
  • Показатель планирования. Тип значения: «Булево». Используется при автоматическом заполнении. Определяет планируемые показатель: количество либо сумма.  
  • Коэффициент роста. Тип значения: «Число». Используется при автоматическом заполнении документа методами наивного прогнозирования  и характеризует рост либо снижение объема закупок в планируемом периоде. При возрастании объема закупок коэффициент больше 1, при снижении объема закупок коэффициент меньше 1.
  • Способ округления. Тип значения: «Булево». Определяет способ округления количества номенклатуры.
  • Коэффициент сезонности. Тип значения: «Число». Используется при автоматическом заполнении документа методами наивного прогнозирования и характеризует рост или снижение объема закупок в планируемом периоде. Данный коэффициент должен характеризовать тенденцию роста или снижения закупок в зависимости от планируемого сезона. Вместе с коэффициентом роста закупок образуется мультипликативная модель прогнозирования, то есть коэффициенты перемножаются. 
  • Количество периодов для анализа. Тип значения: «Число». Количество анализируемых периодов при использовании автоматического заполнения. Размерность периода зависит от реквизита "Периодичность" выбранного сценария планирования. Периоды отсчитываются назад от планируемого периода.
  • Количество периодов смещения. Тип значения: «Число». Количество периодов смещения при использовании автоматического заполнения.
  • Количество сезонов. Тип значения: «Число». Число сезонов.  
  • Модель прогнозирования.  Ссылка на перечисление «Модели прогнозирования». Используется при заполнении методами математического прогнозирования. Определяет используемую математическую модель для обработки временного ряда. 
  • Метод распределения категорий. Ссылка на перечисление «Методы распределения категорий». Используется в случае заполнения номенклатурного бюджета закупок на основании категорийного при распределении суммы категорий по суммам номенклатурных позиций. Определяет способ расчета весовых коэффициентов номенклатуры.  
  • Параметр управления распределением. Тип значения: «Булево». Используется при заполнении номенклатурного бюджета на основании категорийного бюджета. Определяет реквизит табличной части (количество или цена), который будет изменяться при распределении категорийной суммы по суммам номенклатур.
  •  Способ последнего заполнения. Тип значения: «Число». Определяет способ последнего заполнения, необходимо для формирования строки с информацией о последнем используемом автоматическом способе заполнения.  
  • Сумма документа. Тип значения: «Число». Итоговое значение суммы планирования в валюте управленческого учета. 
  • Тип цен. Ссылается на справочник «Типы цен». Тип цен. 
  • Операция. Реквизит является обязательным для заполнения. Ссылается на справочник «Хоз. операции». Выполняемая документом операция. Определяет бюджет планирования (Бюджет закупок по номенклатуре, либо бюджет закупок по категориям).
  • Параметр1. Тип значения: «Число». Используется при автоматическом заполнении методами математического прогнозирования. Определяет степень полинома при прогнозировании полиномиальным трендом или постоянную сглаживающую для данных при прогнозировании методом Хольта-Винтерса. Для остальных математических моделей неопределено.
  • Параметр2. Тип значения: «Число». Используется при автоматическом заполнении методами математического прогнозирования. Определяет постоянную сглаживающую тренда при прогнозировании методами Хольта-Винтерса. Для остальных математических моделей неопределено.
  • Параметр3. Тип значения: «Число». Используется при автоматическом заполнении методами математического прогнозирования. Определяет постоянную сглаживающую сезонности при прогнозировании методами Хольта-Винтерса. Для остальных математических моделей неопределено.

Табличная часть «Товары»

Основная табличная часть документа, предназначенная для хранения запланированных данных.

  • Номенклатура. Имеет составной тип. Ссылается на справочник «Номенклатура», либо на справочник «Типы номенклатуры» , в зависимости от Вида бюджета. Прогнозируемая аналитика.
  • Количество. Тип значения: «Число». Запланированное количество номенклатуры. Используется только при номенклатурном планировании.  
  • Единица. Ссылается на справочник «Единицы измерения». Единица измерения номенклатуры.
  • Цена. Тип значения: «Число». Цена номенклатуры на дату документа, соответствующая выбранному типу цен. Используется только при номенклатурном планировании.
  • Нормативная цена. Тип значения: «Число». Нормативная цена номенклатуры. Рассчитывается из запланированной суммы себестоимости. 
  • Сумма упр. Тип значения: «Число». Запланированная сумма продажи номенклатуры\типы номенклатуры в валюте управленческого учета. 
  • Себестоимость упр. Тип значения: «Число». Запланированная сумма себестоимости номенклатуры\типы номенклатуры в валюте управленческого учета. 
  • Маржинальная прибыль. Тип значения: «Число». Маржинальная прибыль.
  • Ставка НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по номенклатурной позиции.
  • Сумма НДС рег. Тип значения: «Число». Запланированная сумма НДС от продажи в валюте регламентированного учета.

Формы

  • Форма списка. Основная форма списка. Используется для отображения списка или выбора документов данного типа.
  • Форма документа. Основная форма документа. Используется для заполнения обязательных реквизитов.
  • Форма списка для выбора. Основная форма выбора.
  • Форма для заполнения простым способом. Произвольная форма. Предназначена для заполнения документа данными, используя метод наивного прогнозирования. Наивное прогнозирование заключается в получении простого арифметического среднего прогнозируемого ресурса (количество или сумма упр.) от фактических или запланированных данных предыдущих периодов. 
  • форма для заполнения математическими методами. Произвольная форма. Предназначена для заполнения документа данными, используя определенные методы математического прогнозирования. Суть работы заключается в следующем: получается временной ряд для каждой номенклатуры/категории номенклатуры. Временной ряд - в данном случае, значения оборотов ресурса: количества или суммы упр, полученные по периодам. На основании данных временных рядов строятся математические модели. Далее, согласно математическим моделям, находятся прогнозные значение ресурса для каждой номенклатуры\категории номенклатуры.
  • Форма для заполнения по документу основанию. Произвольная форма. Предназначена для заполнения документа данными на основании другого документа «Бюджет продаж». Позволяет производить распределение сумм категорий номенклатуры по суммам номенклатурных позиций. То есть, от общего к частному. 
  • Форма для выбора типа цен. Произвольная форма. Используется для выбора типа цены, и коэффициента пересчета из валюты типа цены в валюту управленческого учета.
  • Результат. Произвольная форма. Используется для графического отображения фактического и смоделированного временных рядов при прогнозировании математическими методами.

Способ вызова

Вызывается из меню "Документы--> Бюджетирование -->Бюджет продаж".

Движения
