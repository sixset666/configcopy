Регистр сведений "Платежи"

Назначение

Регистр предназначен для хранения всех операций по авторизации платежей Cyberplat настроек утверждения документов  в разрезе вида документов, объекта настройки (подразделения или пользователя) и статуса документа.

Относится к подсистеме: «Сервис».

Параметры

Периодичность:Непериодический

Режим записи: Независимый

Измерения

  • Компьютер. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Объект, на уровне которого производится настройка.
  • Касса. Реквизит является обязательным для заполнения. Ссылается на справочник «КассаККМ». .
  • Оборудование. Реквизит является обязательным для заполнения. Ссылается на справочник «Оборудование». Является действием, которое разрешено производить объекту при выполнении утверждения документа.
  • Фискализация.  Признак фискализации чека, 0 - нет, 1 - да
  • Статус. Ссылается на перечисления «СтатусВыполненияДействия». 
  • КодАвторизации. Ссылается на план видов характеристик «Права и настройки». 
  • Параметр1. Тип значения: «Строка». Параметр авторизации платежа
  • Параметр2. Тип значения: «Строка». Параметр авторизации платежа
  • Параметр3. Тип значения: «Строка». Параметр авторизации платежа
  • ДатаЧека. Тип значения: «Дата».  Ссылается на план видов характеристик «Права и настройки». 

Ресурсы

  • Сумма. Тип значения: «Число». Сумма платежа.

Реквизиты

  • НомерЧека. Тип значения: «Число». Номер чека, пробитого на ФР.
  • ВремяЧека. Тип значения: «Дата». Время чека, пробитого на ФР.
  • ФискальныйНомерПозиции. Тип значения: «Дата». Номер строки исходного документа.
  • КодОшибки. Тип значения: «Число». Штрих-код запрещен. Последний код ошибки
  • ОписаниеОшибки. Тип значения: «Строка». Штрих-код запрещен. Описание последней ошибки.
  • ЛогАвторизации. Тип значения: «Строка». Лог попыток платежа, таблица, выгруженная в строку.

Способ вызова

Не предназначен для интерактивного вызова и редактирования.

Дополнительная информация

Установка значений для записи в данный регистр производится из обработки «ФронтКассира» при добавлении платежа в товарную часть.
