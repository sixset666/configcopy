Справочник "Виды платежей"

Назначение

Справочник "Виды платежей" предназначен для ведения списка видов платежей, которые наш магазин может выполнять в пользу операторов сотовой связи.

Наименование вида платежа должно либо полностью соответствовать названию оператора, либо частично повторять  его первые символы.

Относится к подсистеме: «Сервис».

Параметры

Длина кода: 8, тип кода: Строка.

Длина наименования: 30.

Не иерархический.

Реквизиты

  • Авторизатор. Реквизит является обязательным для заполнения. Ссылается на справочник «Оборудование». Устройство-авторизатор платежа.
  • Контрагент. Реквизит является обязательным для заполнения. Ссылается на справочник «Контрагент». Провайдер, предоставляющий услуги сотовой связи.
  • Договор. Реквизит является обязательным для заполнения. Ссылается на справочник «ДоговорВзаиморасчетов». Договор, оговаривающий условия предоставления комиссионной услуги по оплате платежей за сотовую связь.
  • ОбработкаПлатежей. Реквизит является обязательным для заполнения. Тип значения: «Строка». Адрес внешнего файла обработки (*.epf), для обработки параметров.
  • Обработка. Реквизит является обязательным для заполнения. Тип значения: «ХранилищеЗначения». Внешний файл обработки (*.epf), для обработки параметров.

Формы

  • Форма списка. Основная форма выбора группы. Используется для выбора групп.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов.

Способ вызова

Вызывается из меню "Справочники --> Номенклатурные --> Номенклатура".
