Справочник "Настройки доставки сообщений"

Назначение

Предназначен для хранения настроек доставки сообщений различными способами для отдельного узла распределенной информационной базы.

Относится к подсистеме: «Сервис».

Параметры

Длина кода: 8, тип кода: Строка.

Длина наименования: 100.

Не иерархический.

Реквизиты

  • Способ обмена. Тип значения: «Число». Способ доставки/получения сообщений обмена (0 - файловый, 1 - MAPI, 2 - SMTP/POP3, 3 - FTP).
  • Каталог сообщений обмена. Тип значения: «Строка». Локальный каталог сохранения/получения сообщений обмена.
  • Каталог обмена (локальная сеть). Реквизит является обязательным для заполнения. Тип значения: «Строка». Каталог сохранения/получения сообщений обмена по локальной сети.
  • Адрес электронной почты. Тип значения: «Строка». Электронный адрес получателя сообщений обмена.
  • Таймаут. Тип значения: «Число». Время ожидания ответа от почтового сервера (в секундах).
  • Сервер SMTP. Тип значения: «Строка». Адрес почтового сервера для отправки сообщений обмена.
  • Порт SMTP. Тип значения: «Число». Номер порта почтового сервера для отправки сообщений обмена.
  • Пользователь SMTP. Тип значения: «Строка». Имя учетной записи для авторизации на почтовом сервере отправки сообщений обмена.
  • Пароль SMTP. Тип значения: «Строка». Пароль учетной записи для авторизации на почтовом сервере отправки сообщений обмена.
  • Сервер POP3. Тип значения: «Строка». Адрес почтового сервера для приема сообщений обмена.
  • Порт POP3. Тип значения: «Число». Номер порта почтового сервера для получения сообщений обмена.
  • Пользователь POP3. Тип значения: «Строка». Имя учетной записи для авторизации на почтовом сервере получения сообщений обмена.
  • Пароль POP3. Тип значения: «Строка». Пароль учетной записи для авторизации на почтовом сервере получения сообщений обмена.
  • Профиль MAPI. Тип значения: «Строка». Наименование почтового профиля MAPI, используемого для получения и отправки сообщений обмена.
  • Пароль MAPI. Тип значения: «Строка». Пароль для авторизации на почтовом сервере MAPI.
  • Сервер FTP. Тип значения: «Строка». Адрес FTP сервера, используемого для сохранения/получения сообщений обмена.
  • Порт FTP. Тип значения: «Число». Номер порта FTP сервера, используемого для сохранения/получения сообщений обмена.
  • Пользователь FTP. Тип значения: «Строка». Имя учетной записи для авторизации на FTP сервере.
  • Пароль FTP. Тип значения: «Строка». Пароль учетной записи для авторизации на FTP сервере.
  • Каталог обмена (FTP). Тип значения: «Строка». Каталог сохранения/получения сообщений обмена по FTP.
  • Сервер прокси FTP. Тип значения: «Строка». Адрес прокси-сервера FTP.
  • Порт прокси FTP. Тип значения: «Число». Порт прокси-сервера FTP.
  • Протокол прокси FTP. Тип значения: «Строка».  Протокол прокси-сервера FTP.
  • Пользователь прокси FTP. Тип значения: «Строка». Имя пользователя прокси-сервера FTP.
  • Пароль прокси FTP. Тип значения: «Строка». Пароль пользователя прокси-сервера FTP.
  • Пассивный режим FTP соединения. Тип значения: «Булево». Признак использования пассивного режима для соединения с FTP сервером.
  • Вести лог файл. Тип значения: «Булево». Вести лог файл.
  • Каталог лог файла. Тип значения: «Строка». Каталог лог файла.
  • Вести журнал регистрации. Тип значения: «Булево». Вести журнал регистрации.
  • Вести системный протокол. Тип значения: «Булево». Вести протокол событий (регистр). 
  • Архивировать сообщения обмена. Тип значения: «Булево». Флаг необходимости архивации сообщений обмена 
  • Пароль архива при загрузке. Тип значения: «Строка». Пароль файла-архива для загрузки сообщения.
  • Пароль архива при выгрузке. Тип значения: «Строка». Пароль файла-архива для выгрузки сообщения.
  • Фиксировать прием отправку объектов. Тип значения: «Булево». Признак записи в журнал регистрации факта отправки отдельных объектов.

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов.

Способ вызова

Вызывается из меню "Справочники - Структура компании - Настройки доставки"

Дополнительная информация

При хранении в базе нескольких настроек для одной информационной базы в рамках приема/ отправки сообщений только одна из них может являться текущей настройкой. Данная настройка используется при запуске процедур обмена из командной строки.
