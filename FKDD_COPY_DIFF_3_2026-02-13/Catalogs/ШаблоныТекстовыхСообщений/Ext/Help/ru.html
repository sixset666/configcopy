Справочник "Шаблоны текстовых сообщений"

Назначение

В справочник заносится информация о шаблонных строках сообщений, правила их заполнения по реквизитам объекта метаданных и произвольным выражениям.

Относится к подсистеме: «Служебные».

Параметры

Длина кода: 12, тип кода:Строка.

Длина наименования: 50.

Иерархический

Реквизиты

  • Объект метаданных. Строковое представление объекта метаданных из которого происходит заполнение шаблона.
  • Шаблон. Реквизит является обязательным для заполнения. Строка с мнемоническими ссылками на реквизиты объекта метаданных. 

Табличная часть «Соответствие реквизитов»

  • Реквизит. Строка. Имя реквизита для заполнения.
  • Мнемоника. Строка. Мнемоническое представление реквизита. 

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов.
  • Форма выбора объекта метаданных. Используется для отображения объектов метаданных и последующего выбора.
  • Форма выбора реквизита документа. Форма для выбора реквизита объекта метаданных.
  • Форма группы. Форма редактирования группы справочника авторабот.
  • Форма выбора произвольного выражения. Используется для выбора автотекста.

Способ вызова

Вызывается из меню "Все операции--> Справочники--> Шаблоны текстовых сообщений".

Дополнительная информация

  • Кнопка «Автотекст» .  По нажатию кнопки выпадает список, который состоит из данных табличной части «Соответствие реквизитов». Используется для быстрого редактирования шаблона.
  • Кнопка «Добавить действие» .  По нажатию кнопки выпадает список, который состоит из элементов справочника «Действия». Используется для быстрого заполнения табличной части «Соответствие реквизитов».

    • В табличноый части «Соответствие реквизитов» колонка «Имя» имеет составной тип данных («Строка», справочник «Действия»). По нажатию на поле в табличной части, соответствющему колонке «Имя», выпадает список, где предложено пользователю выбрать реквизит объекта метаданных, либо элемент справочника действия. Такая конструкция позволяет обеспечить гибкость использования шаблонов текстовых сообщений.
