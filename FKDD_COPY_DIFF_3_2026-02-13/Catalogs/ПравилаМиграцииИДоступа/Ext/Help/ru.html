Справочник "Правила миграции и доступа"

Назначение

Предназначен для хранения правил миграции документов в доступные узлы распределенной информационной базы

Основное поведение при миграции определяется реквизитом Вариант настройки.

Относится к подсистеме: «Сервис».

Параметры

Длина кода: 8, тип кода: Строка.

Длина наименования: 100.

Не иерархический.

Реквизиты

  • Вариант настройки. Реквизит является обязательным для заполнения. Ссылается на перечисление «Способы миграции объектов». Определяет возможный способ миграции. 
  • Миграция только проведенных. Тип значения: «Булево». Признак миграции только проведенных документов.

Табличная часть «Подразделения информационных баз»

  • Подразделение. Ссылается на справочник «Подразделения компании». Задает подразделение мигрирующего документа.
  • Информационная база. Ссылается на справочник «Структура информационных баз». Задает информационную базу (узел ИБ), в которую может мигрировать документ.
  • Запретить просмотр. Тип значения: «Булево». Запрещает просмотр документа в информационной базе по реквизиту ИнформационнаяБаза для документа, у которого реквизит Подразделение компании совпадает с реквизитом табличной части Подразделение.
  • Запретить изменение. Тип значения: «Булево».  Запрещает редактирование документа в информационной базе по реквизиту ИнформационнаяБаза для документа, у которого реквизит Подразделение компании совпадает с реквизитом табличной части Подразделение. 

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов.

Способ вызова

Вызывается из меню "Справочники - Структура компании - Правила миграции и доступа"

Дополнительная информация

При выборе способа миграции документов и их движений следует руководствоваться следующими положениями:

Полная миграция. Миграция производится во все доступные узлы плана обмена без каких-либо ограничений.

Миграция запрещена. Миграция не производиться, что аналогично галочке "Запретить регистрацию" для документа в составе плана обмена.

Миграция по подразделениям узлов. Миграция производится в узлы планы обмена, для которых в соответствующей информационной базе в табличной части Подразделения задано подразделение, совпадающее с подразделением мигрирующего документа.

Индивидуальная настройка по подразделениям. Миграция производится в узлы планы обмена, в которых подразделение документа совпадает с подразделением, указанным в табличной части Подразделения информационных баз данного элемента справочника «Правила миграции и доступа».

Исходя из сказанного допустимо создание по одному элементу справочника с вариантами настроек "Полная миграция", "Миграция запрещена","Миграция по подразделениям узлов" и произвольного количества элементов с вариантом настройки "Индивидуальная настройка по подразделениям".
