Справочник "Пользователи"

Назначение

Справочник "Пользователи" предназначен для хранения списка пользователей, которым разрешена работа с системой 1С:Предприятие. На справочнике пользователей построена вся система разделения доступа и авторизации.  Каждый элемент этого справочника описывает параметры одного пользователя конфигурации или шаблона набора прав и личных настроек (код соответствует идентификатору пользователя). Они вводятся автоматически при первом входе в систему нового оператора. Справочник предназначен для редактирования только администратором.
Поля справочника хранят информацию о правах доступа пользователя к различным объектам конфигурации и прочие настройки параметров работы с системой.

Информация о всех действиях пользователя при работе с информационной базой отражаются в журнале регистрации. Журнал регистрации вызывается из пункта меню «Сервис» – «Журнал регистрации».

Относится к подсистеме: «Структура компании».

Параметры

Длина кода: 18, тип кода: Строка.

Длина наименования: 50.

Не иерархический.

Реквизиты

  • Имя (идентификатор). Реквизит является обязательным для заполнения. Тип значения: «Строка». Имя входа в систему.
  • Сотрудник. Тип значения:«Булево». Признак того, что пользователь является сотрудником предприятия. При установленом флаге активизируется закладка "Сотрудник", где можно заполнить основные параметры сотрудника компании или перейти к редактированию полной карточки сотрудника, путем открытия элемента справочника «Сотрудники» из одноименного реквизита.
  • Является шаблоном прав. Тип значения: «Булево». Признак того, что элемент является шаблоном прав. 

    Шаблон прав - это набор ролей, прав и настроек пользователя которые могут использоваться для настройки прав другого пользователя.  Для каждого пользователя могут быть заданы определенные «Права и настройки», которые определяются теми ролями, которые назначаются пользователю. Одни и те же роли могут быть назначены нескольким пользователям.

    В то же время один и тот же пользователь может совмещать несколько ролей. Для каждого шаблона прав может быть назначен свой интерфейс. При входе в программу для пользователя будет загружен интерфейс, в соответствии установленными для него шаблоном.  

    При установлении флага становится неактивным выбор шаблонов прав других пользователей. 

  • Доступные роли. Тип значения: «Булево». Список доступных ролей, которые определяют права пользователя на уровне платформы 1С: Предприятие.

  • Аутентификация 1С: Предприятия. Тип значения: «Булево». Признак аутентификации в 1С:Предприятие.
  • Пароль. Тип значения: «Строка». Персональный пароль пользователя.
  • Подтверждение. Тип значения: «Строка». Подтверждение персонального пароля пользователя.
  • Показывать в списке выбора. Тип значения: «Булево». Признак того, что данный пользователь будет показываться в списке выбора при запуске 1С: Предприятие.
  • Аутентификация Windows. Тип значения: «Булево». Признак использования аутентификации Windows.
  • Пользователь. Тип значения: «Строка». Открывает форму выбора пользователя Windows. Системный пользователь в Windows, доступно в случае установки флага "Аутентификация Windows".
  • Основной интерфейс. Тип значения: «Строка». Выпадающий список с доступными интерфейсами.
  • Язык. Тип значения: «Строка». Выпадающий список с доступными языками интерфейсов.
  • Шаблон прав. Ссылается на справочник «Пользователи». Права берутся от выбранного пользователя.
  • Подразделение. Реквизит является обязательным для заполнения. Ссылается на справочник «Подразделения компании». Подразделение компании, к которому относится пользователь.
  • Организация. Реквизит является обязательным для заполнения. Ссылается на справочник «Организации». Организация, к которой относится пользователь.
  • Карточка. Ссылается на справочник «Карточки». Личная карточка пользователя.
  • Код карточки. Тип значения: «Строка». Уникальный штрихкод карточки пользователя.
  • График работы. Ссылается на справочник «Графики работы». График работы, согласно которому работает пользователь.
  • Режим использования оборудования. Тип значения: «Число». Определяет режим использования оборудования в сессии данного пользователя. Актуально при работе в терминальных сессиях и в других случаях, когда запускается несколько сеансов 1С Предприятия на одном компьютере. При значении по умолчанию - "Совместное использование" оборудование рабочего места будет предоставляться для использования другим сессиям и если это первая запущенная копия 1С:Предприятия на данном компьютере, то сессия пользователя становится "серверной" и обращение к оборудованию будет происходить именно через эту сессию до тех пор, пока она запущена. При значении "Неразделяемый доступ", сессия пользователя не будет становиться "серверной" и не сможет предоставлять доступ к оборудованию данного рабочего места другим сессиям. При этом она сможет использовать доступное ей оборудование, например, на других рабочих местах или на своем собственном рабочем месте, но последнее, только если будет одновременно запущена и серверная сессия тоже (другого пользователя). При значении "Запрещено любое использование" все попытки обращения к оборудованию будут заблокированы, и нельзя будет ни использовать, ни предоставлять доступ к оборудованию. Последнее значение имеет смысл устанавливать для сессий, которые никогда не используют торговое оборудование для ускорения запуска системы.
  • Пароль ККМ. Тип значения: «Строка». Пароль ККМ, служит для идентификации пользователя-кассира на оборудовании.
  • Сотрудник. Ссылается на справочник «Сотрудники». Устанавливает связь между пользователем прикладного решения и сотрудником компании.
  • Должность. Ссылается на справочник «Должности». Должность, которую занимает сотрудник.
  • Дата рождения. Тип значения: «Дата». Дата рождения сотрудника.
  • Дата приема. Тип значения: «Дата». Дата приема сотрудника на работу.
  • Сотрудник уволен. Тип значения: «Булево». Признак того, что сотрудник уволен. При установлении флага в поле "Дата увольнения" проставляется текущая дата.
  • Дата увольнения. Тип значения: «Дата». Дата увольнения сотрудника. При введении даты автоматически проставляется флаг "Сотрудник уволен", если он не установлен.
  • ИНН. Тип значения: «Число». Идентификационный номер налогоплательщика.

 

Формы

  • Форма списка. Основная форма списка и выбора. Используется для отображения списка или выбора элементов.
  • Форма элемента. Основная форма объекта. Используется для заполнения обязательных реквизитов. Также форма элемента используется для заполнения параметров пользователя базы и сотрудника, соответствующих данному пользователю компании. При сохранении элемента справочника проверяется имя пользователя и имя соответствующего ему сотрудника. При различии, имя сотрудника изменяется, о чем информирует сообщение. Форма содержит несколько закладок, каждая из которых устанавливает соответствующие параметры пользователя.
    • Закладка "Доступ". Устанавливается вид аутентификации пользователя. Выбор шаблона, интерфейса и языка системы.
    • Закладка "Параметры пользователя". Установка параметров пользователя: принадлежность к подразделению, организации, режим использования оборудования. Отображается название карточки. При нажатии на кнопку открыть поля карточка, открывается форма элемента справочника «Карточки» принадлежащей пользователю. Если карточки у пользователя нет, то создается новая и открывается ее форма элемента.
    • Закладка "Планировщик задач". Предоставляет доступ к регистру сведений «Задачи планировщика». Предназначена для автоматизации выполнения пользователем периодических задач в определенное время. К таким задачам могут относиться: восстановление последовательности документов, завершение работы системы, открытие различных форм, запуск процедур встроенного языка.
    • Закладка "Сотрудник". Предоставляет возможность просмотра и редактирования элемента справочника «Сотрудники» и регистра сведений «Контактная информация» связанных с пользователем системы.  
    • Закладка "Доступ к складам". Предоставляет доступ пользователя на проведение документов по «Складам».
  • Форма выбора пользователя Windows. Предназначена для выбора системного пользователя при установленном признаке пользователя базы "Аутентификация Windows". 

  • Способ вызова

    Вызывается из меню "Справочники --> Структура компании --> Пользователи".
