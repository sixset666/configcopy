Обработка "АРМ Шинный отель"

Назначение

Обработка представляет собой рабочее место мастера по приему и выдаче шин на сезонное хранение автосервиса. Предназначена для оформления «Заявки на хранение шин», а также ввода подчиненных документов.

Относится к подсистеме: «Автоматизированные рабочие места».

Формы

Интерфейс "Шинный отель"

Является основной формой АРМ. Форма состоит из следующих элементов:

  • Панель "Заголовок"
    • Строка поиска. Поиск осуществляется по документам «Заявки на хранение шин». Критериями для поиска служат номер документа, наименование контрагента, наименование автомобиля, государственный номер автомобиля, VIN-код автомобиля, комментарий к документу. Если в результате поиска найдено более одного документа, то будет открыта форма со списком найденных документов. Полученный в результате поиска документ открывается для редактирования в АРМ.
    • Кнопка "Настройка полей поиска" . Служит для указания перечня полей, по которым производится поиск. В форме настройки полей поиска обязательно должно быть выбрано хотя бы одно поле.
    • Меню действий документа. В заголовке меню отображается номер текущего документа «Заявка на хранение шин». Меню содержит следующие действия:

  1. Открыть форму. Открытие формы текущего документа.
  2. Найти в списке. Открытие формы списка заявок на хранение шин и позиционирование на текущем документе.
  3. Печать. Печать выбранной печатной формы.
  4. Выбор печатной формы. Подменю с перечнем доступных печатных форм документа.
  5. Ввести на основании. Создание документа на основании текущего документа и открытие его формы.

  • Область "Клиент и автомобиль"

Содержит основные реквизиты документа:

  • Клиент. Обязательный для заполнения. Ссылается на справочник «Контрагенты и контакты». Клиент, который сдает шины на хранение. Справа от поля ввода расположена кнопка перехода к форме быстрого ввода контрагента.
  • Автомобиль. Ссылается на справочник «Автомобили». Автомобиль клиента, на который установлены шины.
  • Поле "Информация об автомобиле". Содержит подробную информацию об автомобиле.  

  • Область "Параметры расчета"

    • Цены и валюта. Кнопка перехода к форме "Цены и валюта".  Форма используется, когда необходимо изменить для текущего документа следующие реквизиты: валюту, категорию цены, скидку или вид оплаты.
    • Договор.  Обязательный для заполнения. Ссылается на справочник «Договоры взаиморасчетов». Договор, по которому ведутся взаиморасчеты с плательщиком.  
    • Склад. Ссылается на справочник «Склады (места хранения) компании». Склад компании, на котором предположительно будут хранится шины. В дальнейшем в документе «Расходный складской ордер» может быть выбран другой склад. Допускается выбор ордерного склада, использование розничного склада запрещено.
    • Ячейка. Ссылается на справочник «Ячейки хранения». Ячейка склада, в которой предположительно будут храниться шины. Заполняется, когда в правах выбрано хранение шин комплектами. В этом случае все шины в заявке располагаются в данной ячейке.
    • Дата начала. Реквизит является обязательным для заполнения. Тип значения: «Дата». Дата начала хранения шин.
    • Дата окончания. Реквизит является обязательным для заполнения. Тип значения: «Дата». Дата окончания хранения шин.
    • Состояние документа. Состояние заявки. Правила формирования описаны в справочной информации к документу «Заявки на хранение шин».

  • Область "Шины"

Содержит табличную часть "Шины" документа «Заявка на хранение шин». Табличная часть может быть представлена в виде закладок либо в виде таблицы. По кнопке "Заполнить" ячейки на странице могут быть заполнены выбранными значениями шины и диска. При этом в табличной части "Шины" будут аналогично созданы и заполнены 5 строк. Назначение реквизитов в ячейках эквивалентно содержимому табличной части документа. В поле картинки может быть добавлена фотография шины, которая сохраняется в регистре «Картинки и файлы». В верхнем правом углу каждой ячейки расположена кнопка "очистить все поля", которая очищает поля данной шины, а также выбранное изображение.

  • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Шина, принимаемая на хранение.
  • Диск. Тип значения: «ПеречислениеСсылка.ВидыАвтомобильныхДисков». Вид диска, установленного вместе с шиной.
  • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество шин.
  • Единица измерения. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Единицы измерения». Единица измерения шины.
  • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент пересчета по отношению к базовой единице.
  • Расположение шины. Реквизит является обязательным для заполнения. Тип значения: «ПеречислениеСсылка.РасположениеШин». Расположение шины на автомобиле.
  • Характеристика номенклатуры (Серийный номер). Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика шины в зависимости от выбранного способа учета харакетристик для данной номенклатуры (вида номенклатуры). В АРМ поле доступно только при условии, что ведется учет по серийным номерам.
  • Примечание. Тип значения: «Строка». Описание шины, сдаваемой на хранение. В примечание могут быть добавлены значения из справочника «Дефекты» с помощью кнопки выбора. Выбранные в справочнике значения добавляются в примечаение в виде строки.
  • Ячейка. Ссылается на справочник «Ячейки хранения». Ячейка хранения для номенклатуры. Ячейка хранения, в которой предполагается хранение шины. Заполняется, когда в правах запрещено хранение шин комплектами. В этом случае все шины в заявке могут располагаться в различных ячейках.

    • Область "Услуги"

    Содержит табличную часть "Услуги" документа «Заявка на хранение шин».

    • Номенклатура. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Номенклатура». Услуги, оказываемые в рамках заявки по хранению шин.
    • Количество. Реквизит является обязательным для заполнения. Тип значения: «Число». Количество оказываемых услуг.
    • Единица измерения. Реквизит является уникальным и обязательным для заполнения. Ссылается на справочник «Единицы измерения». Единица измерения услуги.
    • Коэффициент. Реквизит является обязательным для заполнения. Тип значения: «Число». Коэффициент пересчета по отношению к базовой единице услуги.
    • Цена. Тип значения: «Число». Цена услуги по хранению.
    • Сумма. Тип значения: «Число». Сумма всегда без скидки. НДС входит в сумму если это указано в типе цен. Сумма всегда равна Цена*Количество.
    • Процент скидки. Тип значения: «Число». Процент скидки на услугу.
    • Скидка. Тип значения: «Число». Сумма скидки по строке документа.
    • Всего. Тип значения: «Число». Итоговая сумма по строке документа со всеми налогами.
    • Процент НДС. Ссылается на справочник «Ставки НДС». Ставка НДС по услуге.
    • НДС. Тип значения: «Число». Сумма НДС по услуге.
    • Характеристика номенклатуры. Реквизит является уникальным. Ссылается на справочник «Характеристики номенклатуры». Характеристика услуги.
    • Партия. Ссылается на план видов характеристик «Типы партий». Партия, с которой списывается товар.

    • Панель основных действий

    Содержит кнопки основных действия и контекстных переходов между АРМ.

    Основные действия:

      • Печать. Печать выбранной печатной формы. 
      • СМС. Ввод на основании документа «SMS».
      • Реализация.  Ввод на основании документа «Реализация товаров».
      • Принять. Ввод на основании документа «Приходный складской ордер».
      • Выдать. Ввод на основании документа «Расходный складской ордер».
      • Новый документ. Создание нового документа.
      • Загрузить. Выбор документа для загрузки в АРМ.
      • Записать и создать новый. Запись и проведение документа (режим проведения - неоперативный). Если запись прошла успешно, создается новый документ и загружается в качестве текущего в АРМ.
      • Записать. Запись и проведение документа.
      • Закрыть. Закрыть форму.

    Способ вызова

    Открывается из обработки "Функциональная панель". Отдельно не используется.
